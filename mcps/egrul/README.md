# egrul-mcp

MCP server для проверки российских контрагентов в ЕГРЮЛ/ЕГРИП через [api-fns.ru](https://api-fns.ru).

Lookup компаний по ИНН/ОГРН, поиск по названию, массовая проверка, **локальная валидация контрольной цифры** (экономит квоту API на невалидных идентификаторах).

---

## Установка

```bash
# Запустить через uvx (рекомендуется)
uvx egrul-mcp

# Или установить и запускать вручную
pip install egrul-mcp
egrul-mcp
```

---

## Подключение к Claude Code

**Шаг 1.** Получи API ключ:
1. Зарегистрируйся на <https://api-fns.ru>
2. В личном кабинете подключи тариф `СПК-старт` (бесплатно, 800 запросов суммарно)
3. Скопируй ключ (40 символов)

**Шаг 2.** Установи переменную окружения:

```bash
export API_FNS_KEY=твой_ключ_из_api_fns_ru
```

**Шаг 3.** Зарегистрируй MCP сервер:

```bash
claude mcp add --transport stdio egrul -- uvx egrul-mcp
```

Или через `.mcp.json` в проекте:

```json
{
  "mcpServers": {
    "egrul": {
      "type": "stdio",
      "command": "uvx",
      "args": ["egrul-mcp"],
      "env": {
        "API_FNS_KEY": "${API_FNS_KEY}"
      }
    }
  }
}
```

## Что умеет

| Tool | Что делает | Использует API |
|------|-----------|----------------|
| `lookup_company(identifier)` | Полная карточка ЮЛ/ИП по ИНН (10/12 цифр) или ОГРН (13/15 цифр) | `/api/egr` |
| `search_companies(query, limit?)` | Поиск по названию / адресу / реквизитам | `/api/search` |
| `batch_check_problematic(identifiers)` | Массовая проверка до 100 ИНН — возвращает **только проблемные** (ликвидированные, в банкротстве) | `/api/multcheck` |
| `validate_identifier(value)` | Локальная проверка контрольной цифры — **без HTTP-запроса** | (локально) |

**Resource:**
- `egrul://identifier-guide` — справка по форматам ИНН/ОГРН/КПП, статусам, red flags при проверке контрагента.

## Примеры использования через Claude

```
>>> Проверь Сбербанк по ИНН 7707083893
[вызывает lookup_company("7707083893")]
→ Возвращает: статус "Действующее", дата регистрации 1991-06-20,
  руководитель Греф Г.О., ОКВЭД 64.19, уставный капитал 67 760 844 000 руб.

>>> Найди все организации со словом "Газпром"
[вызывает search_companies("Газпром", limit=20)]
→ Возвращает список найденных с базовыми реквизитами.

>>> У меня список 50 ИНН поставщиков — проверь кто из них не действует
[вызывает batch_check_problematic([...50 ИНН...])]
→ Возвращает: checked=50, problematic=[...3 проблемных...], invalid=[...].
  Те, кого нет в ответе — действующие, всё ок.

>>> Это валидный ИНН: 1234567890?
[вызывает validate_identifier("1234567890")]
→ {valid: false, kind: "invalid"}.  HTTP запрос не делается.
```

## Локальная валидация — почему важно

api-fns.ru на бесплатном тарифе даёт всего **800 запросов суммарно**. Каждый невалидный ИНН (опечатка, лишняя цифра) — потерянный запрос.

`egrul-mcp` локально проверяет контрольную цифру по официальным алгоритмам ФНС **до** HTTP вызова. Невалидные идентификаторы отсекаются с понятной ошибкой, экономя квоту.

```python
# Локально, без HTTP:
classify_identifier("7707083893")   # "inn-ul"     (Сбербанк, валидный)
classify_identifier("1234567890")   # "invalid"    (неправильная контрольная)
classify_identifier("1027700132195") # "ogrn"      (ОГРН Сбербанка)
```

## Технические детали

- **Источник:** api-fns.ru (коммерческая обёртка над данными ФНС)
- **Транспорт:** stdio
- **Кэширование:** 1 час TTL в памяти процесса, FIFO eviction на 1000 entries
- **Rate limit:** 3 одновременных запроса + ≥400ms между последовательными запросами
- **Timeout:** 30 секунд per запрос
- **Безопасность:** API ключ **никогда** не попадает в логи, error messages, cache keys

---

## Ограничения

- **Бесплатный тариф api-fns.ru — 800 запросов суммарно** (не в день). Для production использования — платный тариф от 10 000 руб/год.
- Альтернатива: **dadata.ru** (10 000 запросов/день бесплатно, header auth). MCP сервер для dadata — на roadmap.
- **Бенефициары** в ЕГРЮЛ есть только прямые учредители — реальные конечные бенефициары (КЦБ) только частично через специальные сервисы.
- **api-fns.ru rate limits** официально не задокументированы. При промышленном использовании рекомендуется тариф "Реквизит" или "Профи".
