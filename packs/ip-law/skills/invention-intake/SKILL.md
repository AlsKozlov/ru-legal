# /invention-intake

## Назначение

Initial intake разработки — assessment IP potential + immediate actions для
preserve rights.

## Pre-flight

- **Disclosure form** заполнена (or filling вместе с разработчиком)
- **Confidentiality** maintained (не publish до filing!)

## Workflow

### Шаг 1. Intake form

```markdown
## Invention / Development Intake

### Basic info

- **ID:** INV-[YYYY-NNN]
- **Title:** [рабочее название]
- **Дата создания / завершения:** [DD.MM.YYYY]
- **Authors / inventors:**
  - Имя 1: [роль, % вклада]
  - Имя 2: ...
- **Worker / contractor / cooperation:** [конкретный статус каждого автора]
- **Department / project:** [...]

### Описание

#### Что это

[Plain English / Russian description]

#### Технические features (для патента)

- Feature 1: ...
- Feature 2: ...
- ...

#### Преимущества над существующими

- Над существующими аналогами: ...
- Прирост эффективности: ...
- Стоимость / простота / etc.: ...

#### Сферы применения

[...]

### Источники + datas + материалы

- Исходные коды / схемы / чертежи: [link / path]
- Black-box experiments / тесты: [data]
- Prior art знание (что мы видели до): [...]
- Сторонние компоненты использованы (OSS / commercial libraries): [list]
```

### Шаг 2. Patentability assessment (preliminary)

#### Для изобретения (ст.1350 ГК)

| Criterion | Met? | Anaylsis |
|-----------|------|----------|
| **Новизна** — неизвестно из уровня техники | [Y/N] | Поиск через ФИПС / EPO / WIPO |
| **Изобретательский уровень** — для специалиста неочевидно | [Y/N] | Не obvious с учётом prior art? |
| **Промышленная применимость** | [Y/N] | Может быть применено в industry / сельхозе / медицине / etc.? |

#### Для полезной модели (ст.1351)

- Новизна + промышленная применимость (без требования "изобретательский уровень")
- Только для устройств (не способов / веществ)
- Срок защиты — max 10 лет

#### Для промобразца (ст.1352)

- Новизна + оригинальность
- Эстетические / эргономические особенности изделия

#### Для ПО / БД (авторские)

- Защита **automatically** с момента создания (без регистрации)
- Регистрация в Роспатент — opt-in, но даёт **значительную доказательную силу**
- IT-аккредитация в Минцифры — для налоговых льгот (по реестру ПО)

#### Для ноу-хау (ст.1465+)

- Конфиденциальность + коммерческая ценность + меры по охране (режим КТ 98-ФЗ)
- Не нуждается в регистрации
- Защищается **только** при сохранении confidentиальности

### Шаг 3. Ownership analysis (critical)

#### Если автор — работник

**ст.1295 (для авторских работ — ПО / контент)** и **ст.1370 (для изобретений)** apply:

##### ст.1295 — авторские служебные произведения

```markdown
## Анализ служебного произведения (ст.1295)

- [ ] Работник работает по трудовому договору
- [ ] Создание произведения **входит в трудовые обязанности** (ТД / должн.инструкция / приказ)
- [ ] Создано в рамках указанных обязанностей (а не личный проект)

→ Если все Y: исключительное право у работодателя по умолчанию.

### Right of return (ст.1295 ч.2-3)

Право возвращается к автору если работодатель **в течение 3 лет** с момента
получения произведения от работника **не начал использование, не передал и не
сообщил автору о решении сохранить в тайне**.

→ Action: каждое intake — записать действие работодателя (начало использования / решение хранить).

### Вознаграждение

- Работник имеет право на отдельное вознаграждение (ст.1295 ч.2)
- Размер — по соглашению; не согласовано → суд
- Лучше: установить в ТД / положении заранее
```

##### ст.1370 — служебные изобретения

```markdown
## Анализ служебного изобретения (ст.1370)

- [ ] Работник работает по трудовому договору
- [ ] Изобретение / ПМ / промобразец создано **в связи с выполнением** трудовых обязанностей или конкретного задания работодателя

### Workflow

1. **Работник** обязан **письменно уведомить** работодателя о создании изобретения (ст.1370 ч.4)
2. **Работодатель в течение 4 месяцев** обязан:
   - (а) Подать заявку в Роспатент для патентования, ИЛИ
   - (б) Передать право на патентование другому лицу, ИЛИ
   - (в) Сообщить работнику о решении сохранить в тайне (с условием выплаты вознаграждения)

3. **Если работодатель в 4 мес ничего не сделал** → право на патентование возвращается к **работнику**

4. **Вознаграждение работнику:**
   - При получении патента работодателем — отдельная выплата
   - При сохранении в тайне — отдельная выплата
   - Размер: по соглашению; не согласовано → fix amount по Правительству РФ (ПП 512/2014)

### Critical action items

- [ ] Запишите дату письменного уведомления от работника
- [ ] Расчёт 4-месячного дедлайна
- [ ] Engagement патентного поверенного для filing если планируем
- [ ] Если решение хранить в тайне — установить режим КТ + договор о неразглашении
- [ ] Вознаграждение — оформление выплаты
```

#### Если автор — исполнитель по ГПД

```markdown
## Анализ договорного IP

- **Договор:** [reference]
- **Тип:** ГПД (договор оказания услуг / подряда)
- **Содержит ли явное отчуждение исключительных прав к нам?**: [Y/N]

→ **Если N**: исключительное право остаётся у исполнителя (это **default** в РФ — НЕ как US work-for-hire!)

→ Action items:
- [ ] Negotiate доп.соглашение к ГПД для отчуждения (ст.1234)
- [ ] Если impossible — получить хотя бы исключительную лицензию (но cost ниже)
- [ ] Проверить договор содержит явное отчуждение для всех future IP
```

#### Если соавторы — mix

- Каждый соавтор имеет **own доля** в исключительном праве (ст.1258 ГК)
- Использование совместного произведения — по соглашению соавторов
- Без соглашения — каждый может использовать самостоятельно для творческих целей, но не коммерчески без consent

→ Critical: для commercial use ensure все соавторы передали права.

### Шаг 4. Recommended next actions

```markdown
## Recommended actions

### Immediate (within 1 week)

- [ ] Confidentiality enforced — все participants signed NDA / соблюдают положение о КТ
- [ ] Disclosure form complete + signed by all inventors
- [ ] Ownership analysis done — все потенциальные disputes identified

### Short-term (1-4 weeks)

- [ ] Patentability preliminary search (paid через патентного поверенного или базовый)
- [ ] Decision matrix:
  - Patent / зарегистрировать ПО / БД: cost vs benefit
  - Хранить в тайне (ноу-хау + 98-ФЗ): cost vs benefit
  - Публиковать (open-sourcing с защитой brand): cost vs benefit
  - Combinaция (e.g., patent core + ноу-хау для production know-how)
- [ ] Engagement патентного поверенного для filing

### Medium-term (1-3 months)

- [ ] Заявка в Роспатент / ФИПС (см. `/rospatent-application`)
- [ ] (Для соавторов) — оформить соглашение об отчуждении к работодателю
- [ ] Вознаграждение работнику-автору оформлено
- [ ] (Для ПО) — IT-аккредитация в Минцифры (если applicable)
- [ ] Update нашего портфеля (см. `/portfolio`)
```

## Что НЕ делает

- Не делает comprehensive patent search (это патентный поверенный)
- Не подаёт заявку в Роспатент (это `/rospatent-application` + патентный поверенный)
- Не drafts документацию для regimes КТ (это `data-protection` / `corporate-law/policy-drafting`)
