# /ip-clause-review

## Назначение

Review IP-related положения в договорах перед подписанием. Categories:
- Договоры разработки (ПО / контента / промышленных образцов)
- Услуги (consulting / маркетинг)
- Лицензионные договоры
- Договоры отчуждения IP
- Подряд / договоры на выполнение работ (НИОКР)
- Услуги поставки с IP компонентами

## Pre-flight

- **Тип договора** identified
- **Кто заказчик / исполнитель** — мы customer или provider?
- **Какой IP создаётся / используется** — ТЗ / патенты / ПО / БД / авторские / ноу-хау
- **Срок и территория** релевантны

## Key РФ frameworks

### A. Способы передачи исключительных прав (ст.1233 ГК)

**Два варианта:**

1. **Отчуждение** (ст.1234) — full transfer of IP к новому правообладателю
2. **Лицензионный договор** (ст.1235) — право использования при сохранении IP у владельца

**Critical:** В договоре **должно быть явно указано** что именно — отчуждение
или лицензия. **Default — лицензия** (ст.1233 ч.3).

### B. Регистрация в Роспатент (ст.1232)

**Обязательна регистрация** для:
- Отчуждение исключительного права на зарегистрированный IP (ТЗ / патент)
- **Исключительная лицензия** на зарегистрированный IP
- Залог исключительного права
- Изменения в реестре правообладателя

**Без регистрации** — договор **недействителен для третьих лиц** (ст.1232 ч.6).

**Не требует регистрации:**
- Неисключительная лицензия
- ПО / БД (даже если зарегистрированы в Роспатент)
- Авторские произведения (нет регистрационной системы)
- Ноу-хау

### C. Служебное произведение (ст.1295 ГК)

**Critical РФ rule:** Исключительное право на служебное произведение по
**умолчанию принадлежит работодателю**, но:

#### Условия отнесения к служебному

1. Произведение создано работником
2. **В пределах трудовых обязанностей** (по ТД / должн.инструкции / приказу)
3. Работодатель **в течение 3 лет** начал использование / передал / решил о хранении

#### Если условия не выполнены

- Право возвращается к автору
- Возможен restitution prior royalties

#### Право на вознаграждение

- Автор имеет **право на вознаграждение** за служебное произведение (ст.1295 ч.2)
- Размер — по соглашению; если не согласован → определяется судом
- **Невыплата** вознаграждения **в течение 5 лет с момента когда работодатель использовал** — может стать основанием для оспаривания

**Critical implications для contract drafting:**
- Должн.инструкции должны conкретно покрывать creation IP
- Trudovoy договор должен содержать § о служебных произведениях
- Положение о служебных произведениях — best practice (separate ЛНПА)
- Вознаграждение — fix amount / formula в ТД / положении
- При увольнении — клозlure outstanding обязательств по IP

### D. Гражданско-правовой договор (ГПД) с физ.лицом-исполнителем

Если IP создаётся не работником, а исполнителем по ГПД:

- **По умолчанию** исключительное право у **исполнителя** (а не у заказчика!)
- Чтобы передать to заказчику — нужно **явно указать** в договоре отчуждение
- ИЛИ лицензия с конкретным scope

**Common mistake:** заказчик платит за разработку ПО по ГПД, но без явного отчуждения → исключительное право остаётся у разработчика. Заказчик имеет только право использования (если оплатил), но не право продавать / лицензировать дальше.

### E. Авторство (моральные права) ст.1265

Автор имеет **неотчуждаемые** моральные права:
- Право авторства (быть указанным как автор)
- Право на имя
- Право на неприкосновенность произведения
- Право на обнародование

Эти права **не передаются** даже при отчуждении исключительных прав. В контрактах:
- Указать обязательство по signified attribution автора (если необходимо)
- Указать допустимость переработки (изменения) — иначе автор может оспорить

### Шаг 3. Risk flags

#### 🔴 Critical

- **Не указан тип передачи** (отчуждение vs лицензия) — default лицензия — может быть not what мы хотели
- **Не предусмотрена регистрация в Роспатент** для исключительных лицензий на зарегистрированный IP → недействительность для третьих лиц
- **Договор ГПД без явного отчуждения** → исключительное право остаётся у исполнителя
- **Служебное произведение без вознаграждения** в трудовом договоре / положении → потенциальные claims автора
- **Нет warranty по infringement** — buyer / лицензиат принимает full risk
- **Полное отчуждение по ТЗ без quality control** — risk dilution бренда

#### 🟡 Medium

- Scope лицензии узкий относительно needs
- Регистрация не предусмотрена (даже если необязательна) — лучше иметь paper trail
- Sub-licensing не явно регулируется
- Срок лицензии короткий relatively к необходимости
- Royalty формула ambiguous

#### 🟢 OK

- Standard provisions с явным указанием РФ норм
- Реалистичная коммерческая структура
- Adequate warranties + indemnification

### Шаг 4. Recommended language fixes

#### Для отчуждения (ст.1234)

```markdown
**П.X.X. Отчуждение исключительного права**

Исполнитель (Автор) передаёт Заказчику в **полном объёме** исключительное право
на [объект IP] (далее — "Результат работ") в соответствии со статьёй 1234
Гражданского кодекса РФ.

Передача исключительного права осуществляется в момент **подписания акта
сдачи-приёмки выполненных работ** [или: в момент полной оплаты].

[Если объект подлежит регистрации в Роспатент:]
Заказчик обязуется зарегистрировать переход исключительного права в Роспатенте
в течение [60] дней с момента передачи.

Стоимость отчуждения исключительного права включена в общую цену договора и
дополнительной оплате не подлежит.
```

#### Для лицензии (ст.1235)

```markdown
**П.X.X. Лицензионное соглашение**

Лицензиар предоставляет Лицензиату **неисключительную / исключительную**
лицензию на использование [объект IP] (далее — "Объект") на следующих условиях:

- **Способы использования:** [конкретный list]
- **Территория:** [...]
- **Срок:** [...] (с даты [...])
- **Лицензионное вознаграждение:** [формула]
- **Право на сублицензирование:** [да / нет]
- **Регистрация в Роспатенте:** [если исключительная — обязательна]

[Quality control clause для ТЗ:]
Лицензиат обязуется обеспечить соответствие качества товаров / услуг, в
отношении которых используется ТЗ, установленным [Лицензиаром стандартам / ТУ].
```

#### Для служебного произведения (ст.1295)

```markdown
**П.X.X. Служебные произведения**

Произведения, созданные Работником в пределах трудовых обязанностей,
предусмотренных трудовым договором и должностной инструкцией, **являются
служебными произведениями** в соответствии со ст.1295 ГК РФ.

**Исключительное право** на служебные произведения **принадлежит
Работодателю** в полном объёме с момента их создания.

**Вознаграждение** за служебное произведение составляет [сумма / % от...] и
выплачивается Работнику [в составе заработной платы / отдельной выплатой] в
момент [...].

Работник имеет право на указание своего имени в качестве автора произведения
(моральное право).

[Если применимо:]
В случае последующего использования произведения за пределами обычной
эксплуатации (например, лицензирование третьим лицам) Работнику дополнительно
выплачивается вознаграждение в размере [%] от полученных Работодателем
доходов.
```
