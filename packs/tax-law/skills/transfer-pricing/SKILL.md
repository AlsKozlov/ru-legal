# /transfer-pricing

## Назначение

Help с compliance по transfer pricing (раздел V.1 НК, действует с 2012, regularly
revised). Это complex area — engage outside-консультанта для material transactions.

## Key concepts

### A. Контролируемые сделки (ст.105.14 НК)

Сделка считается контролируемой если:

#### A1. Между взаимозависимыми лицами (определение в ст.105.1)

**Взаимозависимость возникает при:**
- Прямое / косвенное участие > 25% (по голосам в УК или активам)
- Одно лицо может назначать > 50% органов управления другого
- Лица состоят в семейных отношениях (для физ.лиц)
- Зависимость через цепочку лиц, чьё участие в каждом > 50%
- Иные основания доказывающие фактический контроль

#### A2. Thresholds для применения раздела V.1 (после 2022 пересмотрено)

**Внутрироссийские сделки между взаимозависимыми (после реформ 2022):**

- Сумма доходов по всем сделкам с одним contractor за год > **1 млрд руб** — все становятся контролируемыми
- Один из сторон применяет специальный налоговый режим (УСН) — порог снижен до 60 млн
- Один из сторон применяет налоговую льготу (например, IT, СЭЗ, Арктическая зона) — другой контроль
- Один из сторон — резидент ОЭЗ — контроль
- Иные специальные случаи

**Сделки с лицами, зарегистрированными в "льготных" юрисдикциях** (список оффшоров)

**Сделки в области внешней торговли товарами мировой биржевой торговли** (нефть, металлы, и т.п.) — особый режим

### B. Методы (ст.105.7-105.13)

Для определения "рыночных" цен:

| Метод | Когда применять |
|-------|------------------|
| **Метод сопоставимых рыночных цен (CUP)** | Если есть сопоставимые сделки с независимыми; preferred where возможно |
| **Метод цены последующей реализации (Resale Price)** | Для торговых посредников |
| **Метод "издержки плюс" (Cost Plus)** | Для производителей / поставщиков услуг |
| **Метод сопоставимой рентабельности (TNMM)** | Когда других методов нет; самый часто используемый на практике |
| **Метод распределения прибыли (Profit Split)** | Для сделок с уникальными intangibles или сложным value chain |

**Метод по умолчанию — CUP**, если нет — другие в иерархии.

### C. Уведомление о контролируемых сделках

**Срок:** до **20 мая** года, следующего за отчётным (ст.105.16)

**Что подаётся:**
- Перечень контролируемых сделок (по электронному формату)
- Группировка по контрагентам
- Применённый метод

**Штрафы за непредставление / искажение:**
- 5,000 - 50,000 руб (КоАП / НК ст.129.4)
- + risk доначисления налогов если ФНС проверит и найдёт занижение

### D. Документация по ТЦ (ст.105.15)

**По требованию ФНС** (а не автоматически) налогоплательщик предоставляет:
- Обоснование выбора метода
- Обоснование сопоставимости / отбора сопоставимых сделок
- Финансовые / экономические характеристики сторон сделки
- Description function/risks/assets distribution
- Анализ рыночной цены (с расчётами)

**Срок предоставления:** **30 дней** с получения требования

**Best practice:** готовить документацию **заранее** для material transactions, не по факту запроса (трудно реконструировать после факта).

## Workflow

### Шаг 1. Identify взаимозависимость

```markdown
## Стороны сделки

- **Сторона А:** [name + ИНН]
- **Сторона Б:** [name + ИНН]

## Анализ взаимозависимости (ст.105.1)

| Основание | Проверка | Применимо? |
|-----------|----------|------------|
| Прямое участие > 25% | Сторона А owns Y% Стороны Б | [Y/N] |
| Косвенное участие > 25% (через цепочку) | [analysis] | [Y/N] |
| Назначение > 50% органов управления | [...] | [Y/N] |
| Семейные отношения | [...] | [Y/N] |
| Иной фактический контроль | [...] | [Y/N] |

**Conclusion:** Стороны [взаимозависимы / не взаимозависимы]
```

### Шаг 2. Identify контролируемость (если взаимозависимы)

```markdown
## Анализ по thresholds ст.105.14

- Суммарный доход по сделкам за год: [сумма] руб
- Применяются ли специальные режимы / льготы у любой из сторон: [Y/N]
- Это сделка с резидентом low-tax jurisdiction: [Y/N]
- Это нефть / металлы / другая биржевая торговля: [Y/N]

**Conclusion:** Сделки [контролируемые / неконтролируемые] для целей раздела V.1
```

### Шаг 3. Если контролируемые — определение метода

```markdown
## Анализ применимого метода

### Функциональный анализ

- Функции, выполняемые каждой стороной (производство / распределение / маркетинг / R&D / финансовое сопровождение)
- Активы, используемые (материальные / нематериальные)
- Риски, принимаемые (рыночный / кредитный / инвентаризационный / тех.риск)

### Сопоставимые сделки

- Сделки с независимыми лицами по тому же товару / услуге — есть?
- Внешние сопоставимые (other companies / publicly available info)?

### Выбор метода

| Метод | Применим? | Reason |
|-------|----------|--------|
| CUP | Y/N | Если internal CUP — да; иначе external CUP по биржевым ценам / market data |
| Resale Price | Y/N | Если посредник без значительных функций |
| Cost Plus | Y/N | Если производитель / услуги |
| TNMM | Y/N | Default fallback |
| Profit Split | Y/N | Сложные multi-party transactions |

**Recommended:** [метод]
```

### Шаг 4. Сбор данных для документации

#### A. Описание сделки

- Стороны
- Предмет (товар / услуга)
- Объём
- Цена
- Условия (доставка, оплата, гарантии)

#### B. Функциональный анализ (FAR)

- Functions per сторонe (table)
- Assets per сторонe
- Risks per сторонe
- Conclusion: характеристика каждой стороны (full-fledged manufacturer / contract manufacturer / limited-risk distributor / etc.)

#### C. Анализ сопоставимых

Для CUP / TNMM:
- Search database (Amadeus / Bureau van Dijk / SPARK / РБК)
- Параметры отбора (industry, geography, size, year)
- Список cопоставимых (10-30 obычно)
- Фильтры применённые (отбраковка)
- Финальная выборка

Для TNMM:
- Расчёт рентабельности (operating margin / net margin / ROA / ROCE)
- Inter-quartile range
- Median / quartile в качестве benchmark

#### D. Применение метода

- Расчёт рыночной цены / диапазона рентабельности
- Comparison с фактической ценой / рентабельностью
- Conclusion: соответствие принципу рыночной цены / нет

### Шаг 5. Подготовка уведомления

Электронный формат — TXT/XML по schema ФНС (актуальная версия). Поля:

- Реквизиты налогоплательщика
- Перечень контролируемых сделок:
  - Дата начала
  - Дата окончания
  - Контрагент (ИНН + название)
  - Предмет сделки (тип товара / услуги — классификатор ОКПД)
  - Сумма
  - Применённый метод
- Подпись + дата

### Шаг 6. Если получено требование ФНС о предоставлении документации

```markdown
## Действия

1. Acknowledge получение (запомнить дату — start of 30-day clock)
2. Decision:
   - Документация уже подготовлена → submit в срок
   - Не подготовлена → spawn rapid project; consider запросить продление
   - Решение compromise → submit что есть + объяснение

3. Engage outside-tax advisor если material exposure

4. Documentary package:
   - Cover letter с описанием
   - Документация по ТЦ (full report)
   - Supporting evidence (договоры, акты, иные)
   - Расчёты в форме рабочих файлов (Excel)
```

## Strategic considerations

### A. Pre-emptive documentation

Готовить документацию **до** запроса ФНС:
- Снижает шанс доначисления
- Demonstrates good faith
- Easier чем reconstruction post-факт

### B. Engage outside-консультанта

Material transactions (> 100М руб) — engage outside-консультанта с experience в transfer pricing:
- Big Four (PwC, EY, Deloitte, KPMG — с местным presence)
- Boutique tax advisors (DLA Piper, Pepelyaev, и т.д.)
- Independent consultants

### C. Risk areas в РФ practice

| Pattern | Risk |
|---------|------|
| Royalty / licensing fees (особенно к низконалоговым юрисдикциям) | Часто оспариваются — реальный value transfer? |
| Management / service fees (intra-group) | Substantiate реальность услуг |
| Loans intra-group (interest rates) | Сравнение со ставками банков |
| Sale to связанному distributor с low margin | Часто recharacterized |
| Sale of intangibles (ТЗ, патенты) к related party | High scrutiny, требует valuation |

### D. Post-2022 landscape changes

- Reduce cross-border сделок с traditional jurisdictions (UK, EU)
- Increase сделок с дружественными странами (СНГ, Китай, Индия, ОАЭ)
- Domestic intra-group transactions всё ещё критичны (УСН interplay)
- Налоговое ускорение фокус на:
  - Sub-холдинги с льготами (IT, Сколково, ОЭЗ)
  - Cross-border licensing (для оставшихся cross-border setups)

## Что НЕ делает

- Не подаёт уведомление в ФНС (это бухгалтерия / IT)
- Не делает full TP study (это outside-консультанты)
- Не handles spore по ТЦ — это `/tax-audit-response`

## Why RU-unique

US §482 IRC + Treasury regulations — different framework. РФ раздел V.1 — modeled on OECD TP Guidelines с РФ specifics (взаимозависимость определение, thresholds, electronic submission).

**Правовая основа:**
- НК раздел V.1 (ст.105.1 - 105.25)
- Особенно: ст.105.1 взаимозависимость, ст.105.14 контролируемые, ст.105.7-13 методы, ст.105.15 документация, ст.105.16 уведомление
- ОЭСР TP Guidelines (как методологическая основа)
- Письма Минфина / ФНС с разъяснениями (publication regularly)
