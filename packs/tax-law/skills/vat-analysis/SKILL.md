# /vat-analysis

## Назначение

Help с НДС-вопросами:
- Применимая ставка для конкретной операции
- Применимость льготы по ст.149
- Возмещение НДС из бюджета
- "Разрывы" в цепочках НДС — реакция на запросы ИФНС
- Анализ сделок на риск переквалификации / отказа в вычете

## Key concepts

### A. Объект обложения (ст.146)

- Реализация товаров / работ / услуг на территории РФ
- Передача товаров / выполнение работ для собственных нужд (если расходы не уменьшают прибыль)
- СМР для собственного потребления
- Ввоз товаров на территорию РФ (импорт)

### B. Ставки (ст.164)

| Ставка | Когда |
|--------|-------|
| **0%** | Экспорт (с подтверждением — ст.165), международные перевозки, гостиничные услуги (некоторые), и т.д. |
| **10%** | Продовольственные товары (по перечню), товары для детей, печатные издания, медицинские изделия |
| **20%** (default) | Всё остальное |
| **Без НДС** | Льготы по ст.149 (медицинские услуги, образование, общественный транспорт, услуги по обработке драгметаллов, и т.д. — closed list) |

### C. Льготы (ст.149) — exhaustive list

Important: льготы НЕ применяются если:
- Операция не указана в перечне ст.149
- Налогоплательщик отказался от льготы (можно по ст.149 ч.5 — для определённых льгот, на min 1 год)

### D. Момент определения базы (ст.167)

**По default:**
- Дата отгрузки / передачи **ИЛИ** дата получения предоплаты — что раньше
- Если предоплата → НДС считается с предоплаты + дополнительно при отгрузке (с вычетом ранее уплаченного с предоплаты)

### E. Счёт-фактура (ст.169)

Required:
- Выставлен в течение **5 календарных дней** с даты отгрузки / получения предоплаты
- С 2014 — **только электронная** счёт-фактура через оператора ЭДО
- Содержит все обязательные реквизиты (ст.169 ч.5)
- Подписана уполномоченным лицом

**Без правильной счёт-фактуры** — нет вычета у покупателя.

### F. Вычеты (ст.171-172)

Условия для применения вычета:
1. **Счёт-фактура от поставщика** правильно оформлена
2. **Товар / услуга** приняты к учёту (книга покупок)
3. **Используются** в деятельности, облагаемой НДС
4. **Поставщик реально существует** и реально операция совершена (доктрина "реальности сделки" — ст.54.1 НК с 2017)

### G. Возмещение (ст.176)

Если сумма вычетов > сумма к уплате → возмещение из бюджета.

**Процедура:**
1. Подача декларации с заявлением о возмещении (выделено в декларации)
2. **Камеральная проверка обязательна** (стандарт 3 мес, может быть продлена до 6 мес)
3. **АСК НДС-2** автоматически проверяет цепочку счёт-фактур — ищет "разрывы"
4. По результатам — решение о возмещении / отказе

**Ускоренное возмещение** (для определённых категорий — ст.176.1):
- IT-аккредитованные
- Крупнейшие налогоплательщики
- С банковской гарантией
- Резиденты Сколково и аналогичных

### H. АСК НДС-2 — автоматизированная система контроля

С 2015 ФНС использует **АСК НДС-2** — auto-cross-check всех счёт-фактур.

**"Разрыв":**
- Покупатель показал в декларации сч.фактуру от поставщика
- Поставщик НЕ показал её в своей декларации (не отчитался / отчитался без)
- Система flag-ует cross-mismatch
- ИФНС направляет требование о пояснениях

**Action при разрыве:**
- Документы подтверждающие реальность (договор, акты, doc transport, оплата, переписка)
- Если поставщик "однодневка" / просто не сдал — risk доначисления; можно argue ст.54.1

### I. ст.54.1 НК — общее правило против abuse

С 2017 — новая доктрина: налоговая выгода не считается обоснованной если:
1. Основная цель сделки — не уплата налога (sham transactions)
2. Сделка фактически не совершена (не реальная)
3. Сделка совершена не тем лицом, которое указано в документах

**Practical implications:**
- Сделки с компаниями-"однодневками" → отказ в вычете НДС + по налогу на прибыль
- Сделки с реальным экономическим смыслом, но через subscription через посредников → risk пересмотра
- Должная осмотрительность при выборе контрагентов — обязательна

#### Workflow A: Анализ "разрыва" (АСК НДС-2)

```markdown
## Анализ разрыва — Контрагент X, счёт-фактура № Y от DD.MM.YYYY, сумма Z

### Что говорит ФНС

[Цитата из требования / акта]

### Анализ реальности сделки

1. **Договор:**
   - Подписан DD.MM.YYYY
   - Подписан уполномоченным лицом X (доверенность есть / директор по ЕГРЮЛ)
   - Условия рыночные
   - С указанием конкретного предмета

2. **Исполнение:**
   - Акт / товарная накладная № X от DD.MM.YYYY, подписан обеими сторонами
   - Документы транспортировки (товарно-транспортная накладная)
   - Места хранения, использования (фото / видео если есть)
   - Свидетели — наши сотрудники, имевшие дело

3. **Оплата:**
   - Платёжное поручение / выписка банка
   - Расчётный счёт контрагента реальный, активный

4. **Контрагент:**
   - Дата регистрации, статус "действующий" в ЕГРЮЛ
   - Учёт в ФНС, нет в реестре подозрительных
   - Уставный капитал, директор по ЕГРЮЛ
   - Среднесписочная численность (если 0 — flag)
   - Адрес — реальный (массовая регистрация — flag)

5. **Должная осмотрительность:**
   - Проверка ЕГРЮЛ перед сделкой
   - Запрошены копии учредительных
   - Проверены реквизиты доверенности
   - Личная встреча с представителями
   - Опыт сотрудничества (для повторных контрагентов)

### Вывод

[Реальность подтверждена / unclear / not real]

### Risk assessment

- Probability отказа в вычете: [%]
- Сумма exposure: [руб]
- Recommended action: [keep вычет с защитой / уплатить / искать ratifying документы]
```

#### Workflow B: Возмещение НДС

```markdown
## Pre-flight checklist для возмещения НДС

### A. Substantive checks

- [ ] Все вычеты подтверждены счёт-фактурами правильной формы
- [ ] Все товары / услуги приняты к учёту
- [ ] Использование в облагаемых НДС операциях documented
- [ ] Реальность сделок может быть substantiated

### B. Cleanliness check

- [ ] Контрагенты — реально существуют (ЕГРЮЛ checked)
- [ ] Нет "однодневок" в основных поставщиках
- [ ] Должная осмотрительность proved (документы запроса контрагентов)
- [ ] АСК НДС-2 risk — есть ли вероятные "разрывы"?

### C. Decision

- [ ] Подаём декларацию с заявлением о возмещении: Y/N
- [ ] Используем ускоренное возмещение (по ст.176.1): Y/N — если да, какое основание

### D. Подготовительные действия

- [ ] Книги покупок / продаж выверены
- [ ] Документы по каждому вычету в порядке (договор + первичка + сч.фактура + оплата)
- [ ] Готовы ответы на ожидаемые требования ФНС о пояснениях
- [ ] Информирована командa: возможен запрос документов от ФНС в любой момент в течение 3-6 мес
```

## Common scenarios

### Scenario 1: Поставщик стал "разрывом" в АСК НДС-2

- Risk: отказ в вычете
- Defense: substantiate реальность + должную осмотрительность
- Если безуспешно — payment + потенциальный иск к поставщику (если что-то можно взыскать)

### Scenario 2: Экспорт — нужно подтвердить 0%

- Срок: **180 дней** со дня экспорта (для подтверждения)
- Документы: контракт, копии транспортных, банк.выписка, и т.д.
- Если не подтвердить — НДС 20% с retroactive effect (плюс пеня)

### Scenario 3: Сделка с компанией из недружественной страны

- Может потребоваться разрешение Правкомиссии (Указ 81/2022)
- Расчёты ограничены — через спец.счета или special procedure
- НДС-applicability — same rules, но платёж может быть сложным

### Scenario 4: IT-аккредитованная компания — НДС-льгота

- ст.149 п.2 пп.26 — реализация прав на использование ПО / БД, включённых в реестр Минцифры — освобождена от НДС
- Условия — конкретный регistry record, конкретные viду лицензии

## Что НЕ делает

- Не оформляет счёт-фактуры (это ЭДО система)
- Не подаёт декларации (это бухгалтерия)
- Не возражает в АСК НДС-2 (это система ФНС)

## Why RU-unique

US не имеет federal VAT. Государства имеют sales tax, but it's transaction tax (consumer pays), not multi-stage value-added tax. РФ НДС — multi-stage with input/output reconciliation, complex compliance.

**Правовая основа:**
- НК глава 21 (НДС в целом)
- ст.146 — объект
- ст.149 — льготы
- ст.164 — ставки
- ст.165 — экспорт 0%
- ст.167 — момент базы
- ст.169 — счёт-фактура

- ст.176 — возмещение
