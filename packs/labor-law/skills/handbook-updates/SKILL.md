# /handbook-updates

## Назначение

Workflow для **обновления существующих ЛНПА**. Differs from `/policy-drafting`
которое creates new policy from scratch. Here: working with existing handbook,
plug in new ЛНПА changes, handle процедуру утверждения / ознакомления.

## Pre-flight

- Который ЛНПА обновляется?
- Изменения существенно меняют условия труда? (если да — ст.74 procedure)
- Профсоюз есть в компании? (если да — ст.372 учёт мнения)
- Кто инициатор изменений? (НПА change / business decision / audit findings)

## Workflow

### Шаг 1. Identify scope изменений

**Что меняется?** Classify:

| Тип изменения | Процедура |
|---------------|-----------|
| Косметика (опечатки, references к старым НПА) | ст.8: издать новую редакцию |
| Уточнение без изменения существенных условий | ст.8: издать новую редакцию + ознакомить |
| Изменение существенных условий труда (зарплата, режим, место, должность) | **ст.74: уведомить за 2 мес, согласие или увольнение по ст.77 п.7** |
| Изменения, ухудшающие положение work-ков | ст.8 ч.4: **недействительны** относительно существующих работников |

### Шаг 2. Учёт мнения профсоюза (ст.372 ТК)

**Если профсоюз есть И ЛНПА входит в перечень "учитываемых":**

ЛНПА требующие учёт мнения:
- Правила внутреннего трудового распорядка (ст.190)
- Положение об оплате труда (ст.135)
- Положение о премировании
- Графики сменности (ст.103)
- Перечни должностей с ненормированным рабочим днём (ст.119)
- График отпусков (ст.123)
- Иные по колдоговору

**Процедура (ст.372):**

1. Работодатель направляет **проект ЛНПА + обоснование** в выборный орган профсоюза
2. Профсоюз в течение **5 рабочих дней** даёт мотивированное мнение в письменной форме
3. Если мнение **положительное** → работодатель утверждает ЛНПА
4. Если мнение **отрицательное** или содержит предложения:
   - Работодатель может согласиться → ЛНПА утверждается с учётом
   - Не согласиться → провести консультации (в течение 3 раб.дней)
   - Если не достигнуто соглашение → работодатель **может** утвердить ЛНПА **с**
     протоколом разногласий
   - Профсоюз может оспорить в ГИТ или в суд

**Flag:** если процедура не соблюдена → ЛНПА может быть признан недействительным
(ст.8 ч.4); ГИТ может выдать предписание.

### Шаг 3. Изменение существенных условий труда (ст.74 ТК)

**Если изменение ЛНПА фактически меняет условия ТД работников** (например, новое
положение об оплате снижает премию), запускается ст.74:

#### Условия применения ст.74

- Изменения должны быть **обусловлены изменениями организационных или
  технологических условий труда** (структурная реорганизация, внедрение нового
  оборудования, новой технологии)
- **НЕ работает** для "просто захотелось" или экономии бюджета без org/tech
  обоснования

#### Процедура (ст.74)

1. **Уведомить work-а письменно** за **2 месяца** до планируемого изменения
2. Указать причины + новые условия + право работника отказаться
3. Если работник **согласен** — оформить доп.соглашение к ТД
4. Если работник **не согласен**:
   - Предложить **другую** работу в той же местности (всё вакантное)
   - Если другой работы нет / работник отказался → увольнение по **ст.77 п.7**
     с **выходным пособием 2 нед среднего заработка** (ст.178)
5. При увольнении — все стандартные финальные расчёты в день увольнения

**Risk:** если изменение нельзя обосновать org/tech причинами → суд признает
незаконным; работник восстановлен; СЗП за вынужденный прогул.

### Шаг 4. Ознакомление работников под подпись (ст.22, ст.68)

**Все ЛНПА должны быть** ознакомлены работниками под подпись.

**Для существующих работников при обновлении:**

- В разумный срок после утверждения (обычно 3-5 раб.дней)
- Лист ознакомления — отдельный документ или роспись в книге учёта
- Электронное ознакомление допустимо если ст.22.1 / ст.312.2 — кадровый
  электронный документооборот (КЭДО) внедрён

**Для новых работников:**

- При приёме на работу, **до подписания** ТД (ст.68 ч.3)

**Если работник отказывается** — составляется акт об отказе. ЛНПА всё равно
применяется (если процедура утверждения соблюдена).

### Шаг 5. Edit workflow

#### A. Косметика — issue новой редакции

```markdown
[Title] (редакция от DD.MM.YYYY)
Утверждено приказом № [N] от [DD.MM.YYYY]
[Если профсоюз учитываемый: с учётом мнения профсоюза, протокол № [N]]

[Текст изменений inline или в виде revision tracker]
```

#### B. Material change — формальная процедура

1. Подготовить draft изменений
2. **Если профсоюз** → отправить в выборный орган (ст.372)
3. Подождать 5 раб.дней + integrate мнение если положительное
4. Утвердить приказом
5. **Если ст.74 (существенные условия)** → уведомить work-ков письменно за 2 мес
6. Cumular consent / refusal trail per работник
7. Ознакомить под подпись после вступления в силу
8. Архивировать предыдущую редакцию

### Шаг 6. Output report

```markdown
# Handbook Update — [название ЛНПА]

## Scope изменений

[brief description of what changes]

## Classification

- Существенные условия труда: [меняются / не меняются]
- ст.74 applicable: [Y/N]
- ст.372 учёт мнения профсоюза: [Y/N — какой ЛНПА?]
- Ст.8 ч.4 ухудшение: [нет / да — flag]

## Процедура checklist

### Pre-utvержден

- [ ] Draft подготовлен
- [ ] Согласован с finance / HR / operations
- [ ] (Если ст.372) проект направлен в профсоюз
- [ ] (Если ст.372) получено мнение профсоюза [DD.MM.YYYY]
- [ ] (Если возражения) консультации проведены / протокол разногласий

### Утверждение

- [ ] Приказ об утверждении ЛНПА
- [ ] (Если ст.74) уведомления работникам за 2 мес отправлены [DD.MM.YYYY]
- [ ] ЛНПА publication / availability обеспечена

### Post-utvержден

- [ ] Лист ознакомления подготовлен
- [ ] Работники ознакомлены под подпись (в течение 5 раб.дней)
- [ ] Акты об отказе (если есть)
- [ ] Предыдущая редакция архивирована с датой

## Risks / flags

[list of issues with citations]

## Если ст.74 — handling неcoпределившихся

- Список работников: [N человек]
- Согласия collected: [N]
- Отказы: [N]
- Other vacancies offered to отказавшимся: [Y/N — список]
- Увольнения по ст.77 п.7 — выходное пособие 2 нед СЗП каждому
```

## Что НЕ делает

- Не drafts новые ЛНПА from scratch (это `/policy-drafting`)
- Не replaces юр.экспертизу outside-адв для material changes
- Не handles communications с профсоюзом — это HR / outside-адв
