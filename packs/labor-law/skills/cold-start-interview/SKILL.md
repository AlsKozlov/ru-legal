# /cold-start-interview (labor-law)

## Назначение

Это **первое**, что user должен запустить после установки pack. Walk-through по
PROFILE.md template, заполнение всех load-bearing полей. Без этого:
- `/termination-review` не знает risk posture
- `/hiring-review` не знает industry / template
- `/wage-hour-qa` не знает региональные коэффициенты
- `/leave-tracker` не знает кадровый состав

## Workflow

### Шаг 0. Pre-flight

- Проверь, существует ли `~/.ru-legal/profiles/labor-law/PROFILE.md`
- Если **существует и не [PLACEHOLDER]**:
  > Profile уже заполнен. Хотите revisit конкретную секцию? (run `/customize`)
  > Или re-run полный interview (overwrite)?

- Если не существует / placeholder → начинаем

### Шаг 1. Кто мы

- **Тип работодателя:**
  - Микро (< 15 работников, упрощённая отчётность доступна)
  - Малое (< 100 работников, упрощённая часть ЛНПА)
  - Среднее (100-250)
  - Крупное (> 250)
- **Организационная форма:** ООО / АО / ИП / Государственное / НКО / Иная
- **ОКВЭД основной:** [код]
- **Sector flags:**
  - IT (если применима IT-аккредитация — преференции по найму иностранцев)
  - Финансы (Банк России регулирует HR processes отдельно)
  - Образование (педработники — особенности по ст.331-336 ТК)
  - Медицина (особенности по ст.350)
  - Транспорт (особенности по ст.328-330)
  - Опасные производства (ст.213 — медосмотры)
  - Розничная торговля
  - Иное

### Шаг 2. Юрисдикции (регионы)

Это критично для **районных коэффициентов и северных надбавок** (ст.146-148, 313-327 ТК).

- **Где зарегистрированы:** [город, регион]
- **Где работают сотрудники:** (список регионов)
  - Северные / приравненные (ст.313-327) — какие
  - Чернобыльская зона (закон РФ 1244-1) — да/нет
  - Особенности субъекта (Москва, СПб — региональные доплаты)
- **Удалённые работники (ст.312.1-312.9):** да/нет, какие регионы
- **Иностранные работники:** да/нет, страны origin

Implication:
- Северные надбавки автоматически applied в `/wage-hour-qa` если регион из списка
- Чернобыльская зона — extra leave 14 дн.
- Иностранцы — миграционные обязательства separately

### Шаг 3. Состав персонала

Critical для termination/hiring risk analysis.

- **Численность:** total + по категориям
- **Защищённые категории (количество approximate):**
  - Беременные (ст.261 — увольнение по инициативе работодателя почти запрещено)
  - Женщины с детьми до 3 лет (ст.261)
  - Одинокие матери / отцы с детьми до 14 / 18 (инвалид)
  - Инвалиды (ст.224 — квота, льготы)
  - Несовершеннолетние (ст.265-272 — особый режим)
  - Иностранные граждане (ст.327.1-327.7 — основания для увольнения добавлены)
  - Совместители (ст.282 — особенности расторжения)
- **Члены профсоюзов:** есть / нет, какие профсоюзы
- **Председатели профсоюзных органов / выборные:** ст.374 ТК (дополнительные защиты)

### Шаг 4. Профсоюз и колдоговор

- **Профсоюз есть?** да / нет
- **Если да:**
  - Название
  - Численность членов
  - Председатель ФИО + контакт
  - Коллективный договор есть? (если да — дата заключения, срок действия)
  - Соглашение по отрасли применимо? (если да — какое)

Implication:
- `/termination-review` будет требовать учёт мнения профсоюза по ст.373 для членов
- `/policy-drafting` — учёт мнения профсоюза по ст.372 для новых ЛНПА

### Шаг 5. ЛНПА

- **Где хранятся ЛНПА (правила трудового распорядка, положение об оплате, штатное расписание, должн.инструкции и т.д.):** [путь]
- **Текущие ЛНПА:**
  - Правила внутреннего трудового распорядка — есть / нет / устарели
  - Положение об оплате труда / премировании — есть / нет
  - Положение о персональных данных (152-ФЗ) — есть / нет
  - Положение о коммерческой тайне (98-ФЗ) — есть / нет
  - Положение о служебных командировках — есть / нет
  - График отпусков (текущий год) — есть / нет
  - Положение о дистанционной работе (если есть удалёнщики) — есть / нет
  - Должностные инструкции — comprehensive / partial / нет

### Шаг 6. Severance baseline (соглашение сторон)

При увольнении по соглашению (ст.78 ТК) — какая компенсация по умолчанию?

- **Default:** [е.g. "1 оклад на каждый отработанный год"]
- **Senior employees:** [е.g. "2 оклада на год"]
- **Top management:** [е.g. "negotiated case-by-case, baseline 6 окладов"]
- **Условия для НДФЛ-льготы:** компенсация не более 3 окладов (6 для Крайнего Севера) не облагается НДФЛ (НК ст.217 п.1)

### Шаг 7. Risk posture

- **Консервативный:** обильно flag, все увольнения рекомендованы через соглашение сторон, polices over-compliant с ТК
- **Средний:** flag procedural risks, recommend escalation при threshold (например > 10М руб exposure)
- **Агрессивный:** только critical flags, accept higher litigation risk

Note: ниже определённого уровня (например по сокращению штата) нельзя выставить
агрессивный — это load-bearing constraint.

### Шаг 8. Эскалация

- **HR Director / Head of HR:** ФИО, email
- **CEO / GM:** ФИО, email
- **GC / Head of Legal:** ФИО, email (если есть)
- **Outside-адв ФПА (трудовое право):**
  - Адвокат основной: ФИО, удост.№, контакт
  - Адвокат backup: ...
- **Профсоюзный представитель (если профсоюз есть):** ФИО, контакт

### Шаг 9. Outside counsel bench

Outside-адв specialization breakdown:
- **Трудовые споры (litigation):** [list]
- **Compliance / regulator interaction:** [list]
- **Иммиграция (если иностранцы):** [list]
- **Sectoral (e.g. финансы / медицина):** [list]
