# /policy-drafting

## Назначение

Drafts **новый** ЛНПА from scratch (или с template) с учётом РФ требований.

Differs from `/handbook-updates` (которое обновляет existing). Here: greenfield
drafting based on PROFILE + applicable ТК / Sectoral requirements.

## Типы ЛНПА поддерживаемые

| ЛНПА | Обязательно ли | Источник требований |
|------|----------------|---------------------|
| Правила внутреннего трудового распорядка (ПВТР) | **Да** для всех (ст.189) | ст.189, 190 |
| Положение об оплате труда / премировании | Если оплата сложнее окладной (ст.135) | ст.135, 144 |
| Положение о персональных данных работников | **Да** при обработке ПДн (152-ФЗ ст.18.1) | 152-ФЗ + ТК ст.85-90 |
| Положение о коммерческой тайне | Если режим КТ вводится | 98-ФЗ ст.10-11 |
| Положение о служебных командировках | Рекомендуется (ст.166-168) | ст.166-168 + ПП РФ 749 |
| Положение об отпусках / графиках работы | Если стандартное расписание не подходит | ст.103, 123 |
| Положение о дистанционной работе | Если есть удалёнщики | ст.312.1-312.9 |
| Положение о медосмотрах | Если работы связаны с обязательными медосмотрами | ст.213, приказ Минздрава 29н |
| Положение о СИЗ (средствах индивидуальной защиты) | Если применимо | ст.221 + приказ Минтруда 766н |
| Положение о профобучении | Если есть ученические договоры | ст.196-208 |
| Должностные инструкции | Хотя ТК не требует, без них трудно увольнять / disciplining | ст.21 косвенно |
| Антикоррупционная политика | Обязательно для определённых (273-ФЗ ст.13.3) | 273-ФЗ |

## Workflow

### Шаг 1. Determine какой ЛНПА drafting

Если user сказал "набросай положение об оплате" → understand:
- Базовая структура (оклад / премии / надбавки / районные коэффициенты)
- Profile context (тип работодателя, численность, регионы — см. PROFILE.md)

### Шаг 2. Gather inputs from PROFILE.md

- Тип / размер работодателя
- Регионы (для районных коэффициентов / северных надбавок если положение об оплате)
- Профсоюз есть? (для процедуры утверждения)
- ОКВЭД основной (для sectoral requirements)
- Состав персонала (для protections)
- Существующие ЛНПА (чтобы не дублировать / противоречить)

### Шаг 3. Draft по template

#### Common header для всех ЛНПА

```markdown
# [Название ЛНПА]

**Утверждено приказом № [N] от [DD.MM.YYYY]**
**[С учётом мнения профсоюза — протокол № [N] от [DD.MM.YYYY] — если применимо]**
**Введено в действие с [DD.MM.YYYY]**

## 1. Общие положения

### 1.1. Назначение

[Цель этого ЛНПА]

### 1.2. Сфера действия

Положение распространяется на всех работников [Работодателя], за исключением [...].

### 1.3. Правовая основа

Положение разработано в соответствии с:
- Трудовым кодексом РФ (ст.X, Y)
- [иные применимые НПА]
- Учредительными документами [Работодателя]
```

#### Sample: Положение об оплате труда

См. файл-приложение / generated structure:

```markdown
## 2. Системы оплаты труда

### 2.1. Применяемые системы

В [Работодателе] применяются:
- Повременная (оклад) — для [категории работников]
- Сдельная — для [категории]
- Окладно-премиальная — для [категории]

### 2.2. Оклад

Размеры окладов установлены штатным расписанием. Изменение оклада оформляется
изменением штатного расписания + доп.соглашением к ТД (с учётом ст.74 ТК если
существенно).

## 3. Премирование

### 3.1. Виды премий

- **Ежемесячная премия** — по результатам работы за месяц, до X% оклада
- **Квартальная премия** — по результатам квартала, до X% оклада
- **Годовая премия** — по результатам года, до X% оклада
- **Разовые премии** — за конкретные достижения, выслугу, праздники

### 3.2. Показатели премирования

[KPI / measurable criteria. Critical чтобы было measurable — иначе работник
может оспорить депремирование как произвольное.]

### 3.3. Депремирование

Премия может быть снижена / не выплачена в следующих случаях:
- Применённое дисциплинарное взыскание в премируемом периоде
- Невыполнение показателей премирования
- [иные конкретные основания]

Депремирование оформляется приказом с указанием конкретного основания.

## 4. Доплаты и надбавки

### 4.1. За работу в выходные / праздники (ст.153)

Оплата в двойном размере либо по соглашению — отгул + оплата в обычном размере.

### 4.2. За сверхурочную работу (ст.152)

- Первые 2 часа — в полуторном размере
- Последующие — в двойном

### 4.3. За работу ночью (ст.154)

Не менее 20% оклада / тарифной ставки за каждый час ночной работы.

### 4.4. Районные коэффициенты / северные надбавки (ст.146-148, 315-317)

[Если регион подпадает — конкретные коэффициенты]

## 5. Сроки и порядок выплаты

### 5.1. Сроки (ст.136)

Зарплата выплачивается **не реже чем каждые полмесяца**:
- Аванс — [DD] числа текущего месяца за работу с 1 по 15
- Окончательный расчёт — [DD] числа следующего месяца за работу с 16 по конец

### 5.2. Способ выплаты

На банковскую карту работника (с письменного заявления и указанием реквизитов) /
наличными в кассе.

### 5.3. При увольнении

Полный расчёт — в день увольнения (ст.140).
```

#### Sample: Положение о персональных данных работников

```markdown
## 1. Общие положения

В соответствии с ФЗ-152 "О персональных данных" и главой 14 ТК РФ ("Защита
персональных данных работника"):

## 2. Цели обработки

- Заключение, изменение и прекращение трудовых отношений
- Учёт рабочего времени и оплаты труда
- Обеспечение охраны труда
- Соблюдение трудового, налогового, миграционного законодательства
- [иное]

## 3. Категории обрабатываемых ПДн работников

- ФИО, дата рождения, паспорт, ИНН, СНИЛС
- Адрес регистрации / проживания
- Контактные данные
- Сведения об образовании, трудовой деятельности
- Сведения о состоянии здоровья (только в объёме, необходимом для работы)
- Сведения о семейном положении, детях (для предоставления гарантий)
- Биометрические ПДн (если применимо — например, СКУД с отпечатком — отдельное согласие)
- Иные категории — [перечислить]

## 4. Правовые основания

- Трудовое законодательство (ТК, ФЗ-167 пенсионное, ФЗ-326 ОМС, и т.д.) —
  основание ст.6 ч.1 п.2 152-ФЗ
- Согласие работника — для случаев, выходящих за рамки правовых обязательств
  работодателя

## 5. Доступ к ПДн

Доступ имеют только лица, чьи должностные обязанности этого требуют:
- Руководитель работника (прямой)
- HR (кадровая служба)
- Бухгалтерия — для расчётов
- Юридическая служба — для compliance
- IT — для technical maintenance, без content access
- Иные — по обоснованной необходимости

Список лиц с доступом утверждается отдельным приказом.

## 6. Сроки хранения

- Личные дела работающих — весь период работы + сроки архивного хранения (75
  лет для документов до 2003 / 50 лет после)
- Документы по личному составу (приказы, ТД) — 75/50 лет
- Документы по расчётам с работниками — 6 лет
- Согласия на обработку — 3 года после прекращения обработки

## 7. Права работника

В соответствии со ст.89 ТК и ст.14 152-ФЗ работник имеет право:
- Знать о хранящихся ПДн
- Требовать уточнения / удаления / блокирования при незаконной обработке
- Обжаловать в РКН / суд

## 8. Меры защиты

[Технические + организационные — по 152-ФЗ ст.19, требования ФСТЭК. УЗ-1/2/3/4
по ПП РФ 1119 для систем обработки ПДн.]

## 9. Передача ПДн третьим лицам

[ФНС, СФР, банк (для зарплатных карт), мед.центр (для медосмотров), страховые,
страховщик ОМС, и т.п. — только в объёме, необходимом для задач]
```

### Шаг 4. Pre-publication checklist

- [ ] Все ссылки на ТК / ФЗ актуальны (проверить редакции)
- [ ] Нет противоречий с другими ЛНПА компании
- [ ] Не ухудшает положение работников vs ТК / коллективный договор (ст.8 ч.4)
- [ ] (Если применимо) учёт мнения профсоюза spelled out в header
- [ ] (Если ст.74 — изменение существенных условий) запланирован 2-мес уведомление
- [ ] Procedure ознакомления (под подпись) запланирована

### Шаг 5. Утверждение и ознакомление

1. **Согласовать** с заинтересованными подразделениями (HR, finance, legal, ops)
2. **(Если профсоюз учитываемый)** отправить проект в выборный орган — 5 раб.дней на мнение
3. **Утвердить** приказом руководителя
4. **Ознакомить** работников под подпись в разумный срок
5. **Архивировать** проект + лист ознакомления

## Что НЕ делает

- Не заменяет outside-адв для high-risk ЛНПА (особенно положение о КТ, антикорр.)
- Не handles communications с профсоюзом — это HR
- Не выпускает приказ — это HR
