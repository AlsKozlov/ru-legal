# /tender-eligibility-check

## Назначение

Pre-bid eligibility verification. Differs from `/tender-documentation-review`
(full document analysis) — focus только на eligibility check. Quick decision:
go to bid prep or not.

## Workflow

### Шаг 1. Identify требования

Extract из документации тендера requirements:

#### Общие требования (ст.31 ФЗ-44)

- [ ] Соответствие требованиям к участникам
- [ ] Отсутствие банкротства / ликвидации
- [ ] Отсутствие налоговой задолженности > 25,000 руб
- [ ] Не в РНП за последние 2 года
- [ ] Отсутствие у руководителя / главбуха судимости за экономические / коррупционные преступления (ст.31 ч.1 п.7)
- [ ] Отсутствие конфликта интересов между участником и заказчиком (ст.31 ч.1 п.9)

#### Дополнительные требования (если есть)

Common categories:
- **Опыт** — N исполненных контрактов аналогичной направленности за последние 3 года
- **Опыт** — суммарный объём > X% от НМЦК
- **Финансовые ресурсы** — выручка / активы / прибыль за период
- **Тех.база** — конкретное оборудование, помещения, штат
- **Сертификации** — ISO, отраслевые, сертификаты соответствия товаров
- **Лицензии** — если деятельность лицензируема
- **СРО** — для строительства, проектирования, изысканий

### Шаг 2. Run check per requirement

```markdown
## Eligibility Check

### Общие требования

| # | Требование | Соответствуем? | Документ для подтверждения | Status |
|---|-----------|-----------------|--------------------------------|--------|
| 1 | Не в процедуре банкротства | ✓ | Выписка ЕГРЮЛ (актуальная < 30 дн) | Готов |
| 2 | Налог.задолженность ≤ 25k | [?] | Справка ФНС | Запросить |
| 3 | Не в РНП за 2 года | ✓ | Проверка через ЕИС закупки автоматически | n/a |
| 4 | Отсутствие судимости у рук-ва | ✓ | Декларация (можно self-cert) | Готов |
| 5 | Нет конфликта интересов | ✓ | Декларация | Готов |

### Дополнительные требования

| # | Требование | Соответствуем? | Документ | Status |
|---|-----------|-----------------|----------|--------|
| 1 | Опыт ≥ 3 исполненных контрактов | ✓ (5 за 3 года) | Копии контрактов + актов | Подготовить пакет |
| 2 | Выручка ≥ 100 млн / год | ✓ (150 млн) | Бух.отчётность | Готов |
| 3 | ISO 9001:2015 | ✗ | Сертификат | **MISSING** — времени получить нет |
| 4 | СРО членство | ✓ | Свидетельство | Готов |

### Documentary readiness

- Готовые документы: [N]
- Требуют sub-mit запросов / получения: [N] — timeline [...]

```

### Шаг 3. Decision logic

```markdown
## Decision

### Если все требования выполнены + документы готовы

→ **GO** для bid. Proceed to `/proposal-draft-tech` + `/proposal-draft-price`.

### Если небольшие gaps но remediable до deadline

→ **CONDITIONAL GO**:
- Action items с deadlines
- Re-check after remediation
- Proceed if completed

### Если major gaps (например missing ISO которое 6 мес получить)

→ **NO-GO**:
- Document why
- Future planning: можно получить sertификат к следующим аналогичным тендерам?
- Альтернатива — субподряд через компанию с ISO

### Если есть РНП issue

→ **STOP**:
- 2-year disqualification
- Confirm period — может уже истёк
- Если actively в РНП — engage outside-адв ФПА для possible expungement

### Если конфликт интересов

→ **CRITICAL**:
- Cannot participate
- Document and move on

## Common eligibility traps

### Trap 1: Налоговая задолженность 25,001 руб

Превышение порога на 1 руб — exclusion. Pre-bid: получить актуальную справку ФНС
+ paid любую задолженность до подачи.

### Trap 2: РНП по 1 контракту с другим заказчиком

Часто заказчик включает в РНП за минор нарушение → 2 года out из всех гос.закупок.
Action: Engage outside-адв; обжаловать включение в АС.

### Trap 3: Affiliation с заказчиком

ст.31 ч.1 п.9 — конфликт интересов. Если any связи (бывший работник, родственник
ГД заказчика, и т.п.) → disclosure обязательна. Может attempt — но risk последующего отстранения.

### Trap 4: Опыт по аналогичным — definition

Что считать "аналогичным"? Заказчик может узко интерпретировать (только тот же
ОКВЭД + та же стоимость + те же объёмы). Pre-bid: clarify через запрос
заказчику если есть сомнения.

### Trap 5: Документы должны быть на момент подачи заявки

Не на момент подведения итогов. Если у нас сертификат истекает 5 дней после
подачи заявки — заказчик может отстранить как несоответствующего.

## Что НЕ делает

- Не подаёт документы заказчику
- Не получает sertификаты
- Не handles РНП дела — это outside-адв ФПА
