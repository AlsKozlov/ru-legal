# /tender-documentation-review

## Назначение

Pre-bid review тендерной документации. Output: go / no-go + список risks +
recommendations для bid.

## Pre-flight

- **Источник:** ЕИС закупки (zakupki.gov.ru) / соответствующая ЭТП
- **Тип тендера:** ФЗ-44 / ФЗ-223
- **Способ определения поставщика:**
  - Электронный аукцион
  - Открытый конкурс
  - Конкурс с ограниченным участием
  - Запрос котировок
  - Запрос предложений
  - Закупка у единственного поставщика (для нас обычно not applicable)
  - Электронный конкурс / котировки в электронной форме

## Workflow

### Шаг 1. Базовые параметры

```markdown
## Тендер [номер закупки в ЕИС]

- **Заказчик:** [название + ИНН + сектор (federal / regional / municipal / гос.корпорация)]
- **Регулирование:** ФЗ-44 / ФЗ-223
- **Способ:** [...]
- **Начальная (максимальная) цена контракта:** [сумма]
- **Источник финансирования:** [бюджет / собственные средства / иное]
- **Дата размещения:** [DD.MM.YYYY]
- **Дата подачи заявок:** [DD.MM.YYYY HH:MM] (срок!)
- **Дата подведения итогов:** [DD.MM.YYYY]
- **Дата заключения контракта (примерно):** [DD.MM.YYYY]
- **Срок исполнения контракта:** [...]
```

### Шаг 2. Анализ технического задания (ТЗ)

#### Что проверяем

| Element | Check |
|---------|-------|
| Описание объекта закупки | Достаточно конкретное? Можем выполнить? |
| Технические характеристики | Соответствуют нашему ассортименту / возможностям? |
| Гост / ТУ / сертификаты | У нас есть? / Можем получить? |
| Брендовые требования | "Аналог не допускается" — flag |
| Узкие технические параметры | "Заточенность" под конкретного — flag |
| Сроки поставки / выполнения | Реалистичны для нас? |
| Условия поставки | Логистика возможна? |
| Гарантия | Срок? Условия? |
| Шеф-монтаж / обучение | Если требуется — у нас есть resources? |

### Шаг 3. Анализ требований к участникам

#### A. Общие требования (ст.31 ФЗ-44)

Участник **должен:**
- Соответствовать требованиям к участникам (no exclusions ст.31)
- Не находиться в процедуре банкротства / ликвидации
- Не иметь налоговой задолженности > 25,000 руб
- Не быть в РНП за последние 2 года
- Не иметь определённых судимостей у руководства (ст.31 ч.1.1)

Самопроверка:
- [ ] Текущая выписка ЕГРЮЛ актуальна (< 30 дней)
- [ ] Налоговая задолженность check (получить справку ФНС)
- [ ] Проверка через РНП (см. `/rnp-check`)
- [ ] Документ об отсутствии судимостей у руководства

#### B. Дополнительные требования (ст.31 ч.2)

Для определённых ОКВЭДов / категорий контракта могут быть:
- Опыт работы (количество лет, аналогичные контракты исполненные)
- Финансовые ресурсы (выручка, активы)
- Тех.базы (оборудование, штат, лицензии)
- Сертификации (ISO 9001, отраслевые)

→ Соответствуем? Документы для подтверждения готовы?

#### C. Лицензии / СРО (для определённых работ)

- Строительство — членство в СРО
- Аудит — лицензия
- IT-services (особенно для гос.заказчиков с ИБ requirements) — лицензия ФСТЭК / ФСБ
- Прочие отраслевые

### Шаг 4. Анализ обеспечения

#### A. Обеспечение заявки (ст.44 ФЗ-44)

- Размер: 0.5-5% от НМЦК (некоторые случаи — выше)
- МСП льгота: 1-5%, не менее 1% от своей предлагаемой цены
- Форма: денежные средства (на спец.счёт ЭТП) либо банковская гарантия (для НМЦК > 20 млн)

#### B. Обеспечение исполнения (ст.96 ФЗ-44)

- Размер: 5-30% от НМЦК (стандарт)
- Антидемпинговые добавки (если bid > 25% ниже НМЦК): + ½ NMCK сверх стандартного
- МСП льгота: можно представить без обеспечения если есть опыт исполнения аналогичных
- Форма: банк.гарантия либо денежные средства

#### C. Обеспечение гарантийных обязательств (ст.96 ч.2.2)

- Optional для заказчика — может потребовать до 10% НМЦК
- Активируется после исполнения контракта на период гарантии

→ У нас есть финансовая возможность? Банк-партнёр готов выдать гарантию?

### Шаг 5. Анализ формы заявки

#### A. Первая часть заявки (для аукциона)

- Согласие с условиями контракта
- Конкретные показатели товара (если требуется) — точно как в ТЗ
- Иные сведения

#### B. Вторая часть заявки

- Сведения о участнике (ИНН + реквизиты)
- Документы подтверждающие соответствие требованиям (опыт, ресурсы, сертификаты)
- Декларации (об отсутствии в РНП, об отсутствии налог.задолженности, и т.д.)

#### C. Конкурс — другая структура

- Заявка с предложением (техническим + ценовым)
- Документы

### Шаг 6. Анализ проекта контракта

#### Особенно опасные пункты

- **Цена твёрдая** (no escalation clauses) — risk если сроки исполнения длинные
- **Сроки исполнения** — нереально короткие → штрафы за просрочку
- **Штрафы / пени** — 1/300 ставки ЦБ + специальные штрафы
- **Гарантийные обязательства** — слишком длинные / широкие
- **Качество** — если "иначе не принимаем" → recommendation подтверждения качества заранее
- **Условия оплаты** — пост-оплата vs. аванс; для МСП — обязательная авансовая часть
- **Расторжение** — основания + последствия

#### Изменения контракта (ст.95 ФЗ-44)

- Ограниченные основания
- В целом не более 10% от цены / объёма
- Не работает без официального соглашения сторон с подписанием

#### Анти-демпинг (ст.37 ФЗ-44)

- Если bid > 25% ниже НМЦК:
  - Обеспечение исполнения удваивается, **ИЛИ**
  - Предоставить обоснование с документами (расчёт что эта цена реалистична)

### Шаг 7. Identification of "заточенности"

См. `/biased-tender-detection` для detailed framework. Quick checks:

- Бренд конкретного производителя в ТЗ без "или эквивалент"
- Узкие технические параметры, которые соответствуют только одному поставщику
- Совокупность параметров, делающая market 1-2 supplier-а
- Заказчик и потенциальный bidder — affiliated (проверить через ЕГРЮЛ, ЕФРСБ)
- Низкое НМЦК (заведомо ниже рынка) — отсеивающее реальных bidders

### Шаг 8. Анализ scoring (для конкурсов)

В конкурсах — не только цена, но и:
- Качество товара / работ
- Опыт исполнения
- Квалификация persona
- Другие критерии

Веса критериев — указаны в документации. Если scoring сильно весит сценарии где мы слабы — risk loss.

## Что НЕ делает

- Не подаёт заявку через ЭТП (это сам участник)
- Не оформляет банковскую гарантию (это банк)
- Не handles ФАС жалобу — это `/fas-complaint-draft`

## Why RU-unique

ФЗ-44 / ФЗ-223 — specific РФ procurement framework. US — FAR (Federal Acquisition Regulation), EU — EU Procurement Directives, в каждом своя структура. РФ — strongly regulated, ЕИС centralized, ФАС oversight.

**Правовая основа:**
- ФЗ-44 «О контрактной системе» (полностью)
- ФЗ-223 «О закупках товаров, работ, услуг отдельными видами юр.лиц»
- ПП РФ № 99 — антидемпинговые меры
- ПП РФ № 1352 — особенности для МСП
- ПП РФ № 815 — типовой контракт
- Постановления ФАС (judicial-like решения по жалобам)
