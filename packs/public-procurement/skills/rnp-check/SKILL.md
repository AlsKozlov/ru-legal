# /rnp-check

## Назначение

Quick check существенный для:
1. **Pre-bid eligibility** — мы не в РНП?
2. **Pre-deal due diligence** — контрагент / партнёр не в РНП?
3. **Reputation check** — для commercial relationships
4. **Subcontractor screening** — субподрядчики на гос.заказ tender

## Pre-flight

- ИНН или полное наименование для проверки
- Доступ к ЕИС закупки (zakupki.gov.ru → раздел РНП)

## Workflow

### Шаг 1. Search РНП

#### Где искать

- **ЕИС закупки** — https://zakupki.gov.ru/epz/dishonestsupplier/quicksearch/search.html
- Параметры поиска: ИНН / название / город

#### Что искать

| Поле | Значение |
|------|----------|
| Наименование контрагента | exact name |
| ИНН | ИНН |
| Дата включения | начало 2-year period |
| Дата окончания | окончание 2-year period |
| Причина включения | конкретное нарушение |
| Информация о заказчике | заказчик инициировавший |
| Реквизиты решения ФАС / суда | для возможного оспаривания |

### Шаг 2. Анализ результата

#### Случай A: Не в РНП

- Output: "Не найден в РНП на дату [DD.MM.YYYY]"
- Save screenshot / выписку для документации (для bid eligibility cert)

#### Случай B: В РНП (текущая запись активна)

- **Дата окончания периода** — критично
- Если до окончания > [our нужная дата] — disqualified для гос.закупки
- Reputation risk для commercial deals (другие могут отказать)

#### Случай C: В РНП, но period истёк

- Запись остаётся в historical РНП
- Не disqualified для текущих гос.закупок
- Но reputation issue остаётся

### Шаг 3. Если мы в РНП (наша компания)

#### Immediate actions

1. **Обжалование** — если попадание неправомерное:
   - **30 дней с момента вкл в РНП на обжалование решения ФАС в АС** (ст.198 АПК)
   - Engage outside-адв ФПА срочно
2. **Mitigation** во время periodа:
   - Communications со стейкхолдерами (заказчики commercial — что произошло)
   - Subcontracting через не-РНП компании (если business model позволяет)
3. **После окончания periodа** — restoration:
   - Не automatic deletion; ждать срок
   - Compliance review всех processes чтобы не повторилось

#### Реабилитация (ст.104 ч.7 ФЗ-44)

В **исключительных** случаях можно request об исключении из РНП до истечения 2-х лет:
- Доказать что причина исключения устранена (например, исполнено судебное решение)
- Восстановление repэeptации добросовестно
- Заявление в ФАС с обоснованием
- Решение ФАС — discretionary

### Шаг 4. Если контрагент в РНП

#### Decision matrix

| Сценарий | Action |
|----------|--------|
| Subподрядчик в РНП на гос.тендер | NOT allowed — заказчик может отстранить нас |
| Commercial поставщик в РНП | Reputation risk; engage carefully; reps & warranties enhanced |
| Контрагент в РНП за минор / спорное нарушение | Document explanation; document due diligence done |
| Контрагент в РНП за major (мошенничество, неисполнение крупного) | High risk; recommend reject |

## Связанные данные

### РНП ФЗ-44 vs ФЗ-223

- ФЗ-44 РНП — основной, ведётся ФАС
- ФЗ-223 РНП — также ведётся ФАС, но separately
- Иногда компания в одном но не в другом
- Check both!

### Другие реестры risk-related

- **Реестр недобросовестных подрядчиков** в строительстве (профильный)
- **Реестр поставщиков-нарушителей контрактов** в обороно-промышленном комплексе
- **Black-lists** конкретных гос.заказчиков (informal — не public)

### Через automation

- API ЕИС закупки доступен — можно автоматизировать regular checks
- СПАРК-Маркетинг агрегирует РНП + другие реестры
- Контур.Фокус, Casebook — commercial services для DD

## Что НЕ делает

- Не обжалует включение в РНП — это outside-адв
- Не делает comprehensive DD — это `corporate-law/diligence-issue-extraction`
- Не отвечает на запрос ФАС в рамках проверки — это outside-адв
