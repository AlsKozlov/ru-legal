# /proposal-draft-tech

## Назначение

Draft technical proposal. Critical для:
- **Электронный аукцион** — техническая часть (первая часть заявки)
- **Конкурс** — техническая часть со всеми параметрами
- **Запрос предложений** — техническое + финансовое предложения

## Pre-flight

- ТЗ внимательно прочитано
- `/tender-eligibility-check` пройден (мы applicable)
- `/tender-documentation-review` сделан (нет "заточенности" блокирующей)

## Workflow

### Шаг 1. Extract from ТЗ

```markdown

### Объект закупки

[Дословно из ТЗ]

|---|----------------|----------------|--------------------------------------|
| 1 | Длина | 100 см | "100 см" exact или "не менее 100" |
| 2 | Материал | Сталь нерж. 12Х18Н10Т | "12Х18Н10Т" or "аналог" |
| 3 | Производитель | "По выбору поставщика" | свобода выбора |
| ... | | | |

### Документы подтверждающие соответствие

- Технические условия (ТУ)
```

### Шаг 2. Apply наш ассортимент

Match: что из нашего портфолио соответствует ТЗ?

- Точное соответствие — best case
- Аналог (если ТЗ allows) — fallback
- Не соответствует — STOP, либо предложить альтернативу либо отказаться

### Шаг 4. Quality check

#### Critical errors (avoid!)

- **Отклонение от ТЗ** в любую сторону без "не менее" / "не более" — отстранение
- **Двусмысленность** в параметрах — заказчик может interpret в нашу пользу или против
- **Опечатки** в тех.характеристиках — отстранение
- **Нет согласия с условиями** (для аукциона) — отстранение
- **Не подписано** электронной подписью — отстранение

#### Common mistakes

- **Использование "около", "примерно", "ориентировочно"** — недостаточная конкретность
- **Несоответствие** между текстом и приложениями (например, сертификат на 100 см а в заявке указано 102) — flag
- **Не подкреплено документами** — например, "имеем опыт" без перечня контрактов
- **Brand confusion** — указали бренд X в заявке, а в документах бренд Y

#### Strategic considerations

- **Match exactly** if ТЗ requires точное значение (12Х18Н10Т)
- **Use upper end of allowed range** if "не менее" — strengthens
- **Don't disclose more than required** — proprietary info
- **Лимит "конкретных показателей"** в первой части аукциона — точно как в ТЗ

### Шаг 5. Submission

- Через ЭТП (через ЛК)
- В рамках срока (до second of deadline — не последний день, не последний час!)
- Электронная подпись активна
- Файлы — в формате, требуемом ЭТП (PDF + .docx обычно)

## Что НЕ делает

- Не подаёт через ЭТП (это сам участник)
- Не оформляет банковскую гарантию (это банк)
- Не handles обжалования отстранения — это `/fas-complaint-draft`
