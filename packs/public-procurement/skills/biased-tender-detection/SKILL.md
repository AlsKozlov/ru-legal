# /biased-tender-detection

## Назначение

Identify "заточенность" в гос.тендере. "Заточенный" = условия специально
сформулированы так, чтобы конкретный поставщик победил. Это нарушение принципов
конкурентной закупки (ст.6 ФЗ-44).

## Indicators of bias

### A. Технические показатели

#### A1. Brand exclusivity без "или эквивалент"

Указание конкретной торговой марки / производителя без "или эквивалент" — **прямое нарушение** ст.33 ч.1 п.1 ФЗ-44.

**Exceptions** (когда можно указать конкретный):
- Уникальный товар, не имеющий аналогов (rare)
- Совместимость с уже имеющимся оборудованием заказчика (must be обосновано)
- Лекарственные препараты по конкретному МНН не работают — указывается МНН

#### A2. Узкие технические параметры

| Pattern | Indicator |
|---------|-----------|
| Длина: 100.0 ± 0.0 мм | Exclusionary — никто другой не сможет |
| Длина: 100.5-105.5 мм | Подозрительно — это специфичный диапазон |
| Длина: 100-150 мм | Normal — широкий range |

**Test:** Search в каталогах — сколько производителей соответствуют каждому параметру?
- Если 1 — точно заточенность
- Если 2-3 — почти точно (или ограниченная конкуренция, что само по себе bias)
- Если > 5 — нормально

#### A3. Совокупность параметров

Даже если каждый параметр сам по себе reasonable, **комбинация** может exclude всех кроме одного.

Например:
- Длина 100 ± 5 мм (~20 производителей)
- × Материал 12Х18Н10Т (~15 производителей)
- × Класс прочности B-IV (~10 производителей)
- × Поверхность зеркальная (~5)
- × Тех.условия конкретного завода (~1) → match только с этим заводом

#### A4. "Эксклюзивные" сертификации

Требование сертификата, который имеет только один производитель — наиболее очевидное.

### B. Сроки

#### B1. Слишком короткие сроки на подготовку

Если сроки между размещением и подачей минимальные (7 дней — минимум по 44-ФЗ для аукциона; меньше — нарушение), это может indicates pre-arranged поставщика с готовыми документами.

#### B2. Слишком короткие сроки исполнения

Если ТЗ требует поставку / выполнение в крайне tight сроки — только тот кто уже готов сможет:
- Поставщик имеет товар на складе
- Поставщик уже субподрядно работает на объекте
- Заказчик заранее предупредил поставщика

### C. Требования к участникам

#### C1. Опыт по конкретным крайне узким направлениям

"Опыт исполнения договоров по проектированию реакторов FAR — Final Adsorption Reactor — минимум 3 contractа за последние 2 года". Если такого опыта в РФ только у одной компании → bias.

#### C2. Финансовые требования избыточные

"Годовая выручка ≥ 5 млрд руб" для small-scope тендера — пытается исключить small players и оставить только incumbent.

#### C3. Местоположение / география

"Производство товара на территории Калининградской области" — без обоснования необходимости. Может exclude legitimate производителей из других регионов.

### D. Цена (НМЦК)

#### D1. НМЦК искусственно занижен

Заведомо ниже рынка → отсеивает реальных bidders → leaves только тех кто готов работать в убыток (часто incumbent с long-term relationships или с suboptimal качеством).

#### D2. НМЦК искусственно завышен

Может также indicate corruption — поставщик ожидает overpricing и kickback к decision-maker'у.

### E. Структурные indicators

#### E1. Identification of вероятного победителя через ЕИС

Через ЕИС закупки можно посмотреть:
- Кто из конкурентов размещает интересующие тендеры по похожим предметам с этим же заказчиком в прошлом
- Сколько win raтe у конкретного поставщика по этому заказчику

**Pattern:** Один поставщик постоянно выигрывает у одного заказчика — flag.

#### E2. Affiliation между заказчиком и потенциальным поставщиком

Через ЕГРЮЛ:
- Совпадение учредителей
- Совпадение директоров (даже в прошлом)
- Bывшие сотрудники заказчика — учредители у поставщика
- Same physical address — может indicate shell schemes

#### E3. Сложная многоступенчатая documentation

Если документация documenation includes very specific clauses, sub-clauses, exhibits, требуемые от участника очень специфические форматы — может indicates что только pre-coached поставщик знает что и как ответить.

### F. Языковые indicators

- ТЗ formulated в крайне сложном "юридическом" стиле — может indicate pre-conditioned поставщик
- Опечатки / inconsistencies в ТЗ — может suggest hasty drafting (less analyzable)
- Cтранные фразы которые "только инсайдер поймёт"

## Workflow

### Шаг 1. Run all checks

```markdown
## "Заточенность" assessment — тендер [номер]

### A. Технические показатели

- [ ] Brand exclusivity (без "или эквивалент"): [Y/N + где]
- [ ] Узкие параметры (доп. без обоснования): [Y/N + список]
- [ ] Combination exclusionary: [Y/N + анализ]
- [ ] "Эксклюзивные" сертификации: [Y/N]

### B. Сроки

- [ ] Подача < 7 дней: [Y/N — если Y то нарушение 44-ФЗ для аукциона!]
- [ ] Исполнение слишком короткое: [Y/N]

### C. Требования к участникам

- [ ] Опыт по крайне узким направлениям: [Y/N]
- [ ] Финансовые требования избыточные: [Y/N]
- [ ] География exclusionary: [Y/N]

### D. Цена

- [ ] НМЦК искусственно занижен (сравнение с рынком): [Y/N]
- [ ] НМЦК искусственно завышен: [Y/N]

### E. Структурные

- [ ] Win history конкретного поставщика у этого заказчика: [N побед за 3 года / общее число тендеров]
- [ ] Affiliation между заказчиком и suspect bidder: [Y/N]
- [ ] Сложная documentation suggesting pre-coaching: [Y/N]

### F. Языковые

- [ ] Странности в ТЗ: [Y/N]
```

### Шаг 2. Probability score

```markdown
## Score

- Total indicators triggered: [N]
- Total possible: [M]
- Bias probability:
  - 0-2 indicators: Low — нормальный тендер
  - 3-5 indicators: Med — подозрительно
  - 6+ indicators: High — almost certainly biased
```

### Шаг 3. Recommendation

#### Low bias

→ Participate если eligibility OK. Standard strategy.

#### Med bias

→ Decision point:
- Если у нас strong position — participate, может быть competitive
- Если ресурсы limited — skip и focus на тендеры с lower bias
- Можно подать запрос разъяснений документации (ст.42 ФЗ-44 — заказчик должен ответить в 2 дня) — если ответ confirms bias → жалоба в ФАС

#### High bias

→ Options:
1. **Skip** — не тратить ресурсы
2. **Жалоба в ФАС** до окончания приёма заявок (см. `/fas-complaint-draft`)
   - Если успешна — тендер останавливается / изменяется
   - Если безуспешна — тендер проходит как был
3. **Participate "anyway"** — иногда biased тендер всё равно открывается для general competition если incumbent failsуществляет (e.g. РНП попадание)

### Шаг 4. Strategic option: запрос разъяснений

Можно подать запрос разъяснений до окончания приёма заявок. Заказчик обязан ответить в 2 рабочих дня (для аукциона / конкурса) или в 3 рабочих дня (для запроса предложений).

**Можно спросить:**
- "Уточните производителя/бренд для соответствия параметру [X]?"
- "Допустим ли эквивалент по параметру [Y]?"
- "Каково обоснование требования об опыте [Z]?"

Ответ заказчика становится **частью документации** и обязателен для всех участников. Это может:
- Раскрыть bias
- Заставить заказчика expand acceptable parameters
- Дать вам пакет documentation для жалобы в ФАС если bias подтверждается

### Шаг 5. Output

```markdown
# Bias Detection Report — [тендер]

## Score: [LOW / MED / HIGH] bias probability

## Triggered indicators

| Indicator | Triggered? | Detail |
|-----------|-----------|--------|
| Brand exclusivity | Y | "Поставка реактора FAR-100 марки Atomic Inc." без "или эквивалент" |
| Узкие параметры | Y | 5 параметров с .0 ± .0 tolerance |
| Узкий опыт | Y | "Опыт исполнения 3+ контрактов по реакторам FAR" |
| Win history | Y | ООО "Атомик-Регион" — 8 побед за 2 года у этого заказчика |
| Affiliation | Unknown | Не нашли direct связи через ЕГРЮЛ, но требует deeper investigation |

## Recommendation

[Skip / Запрос разъяснений → жалоба в ФАС / Skip and document для будущих тендеров от этого заказчика]

## If жалоба в ФАС

→ Use `/fas-complaint-draft`

Срок: до окончания приёма заявок (для предupdate документации) либо в течение 10 дней после подведения итогов (для отмены результатов)

## Documentation для future reference

- Save tender doc + наш analysis
- Pattern recognition for future tendrs от этого заказчика
- Add to internal "blacklist" / "watch list"
```

## Что НЕ делает

- Не подаёт жалобу в ФАС (это `/fas-complaint-draft`)
- Не делает investigation affiliation deep — это могут разные services (СПАРК, ЕГРЮЛ)
- Не handles процесс рассмотрения в ФАС
