# /beneficial-owner-check

## Назначение

Identify бенефициарных владельцев (БВ) юр.лица. Required для:
- 115-ФЗ compliance (KYC юр.лиц)
- 215-ФЗ — публичное раскрытие бенефициаров (с 2022)
- НК ст.93.3 — декларация о бенефициарах в ФНС
- Banking KYC при открытии счёта
- Pre-deal DD (см. `corporate-law/diligence-issue-extraction`)

## Definition бенефициарного владельца (115-ФЗ ст.3)

Физ.лицо, которое:
- **Прямо или косвенно** (в т.ч. через цепочку лиц) **владеет** более **25%** в капитале юр.лица **ИЛИ**
- **Контролирует действия клиента**, в т.ч. имеет возможность контролировать через формальные / неформальные механизмы

**Ключевые элементы:**
- "В конечном счёте" — даже если direct owner — другое юр.лицо, нужно дойти до физ.лица
- "Иначе контролирует" — substance over form

## Workflow

### Шаг 1. Прямые участники

```markdown
## Юр.лицо A: [название + ИНН]

### Прямые участники (по ЕГРЮЛ)

| # | Участник | % доли |
|---|----------|--------|
| 1 | ООО "Альфа" (ИНН XX) | 60% |
| 2 | Иванов И.И. | 30% |
| 3 | ООО "Бета" (ИНН YY) | 10% |
```

### Шаг 2. Расшифровка cascade — куда указывает каждый юр.лицо-участник

```markdown
### ООО "Альфа" (60% юр.лицо A)

#### Участники

| # | Участник | % |
|---|----------|---|
| 1 | Петров П.П. | 50% |
| 2 | АО "Гамма" (ИНН ZZ) | 50% |

#### АО "Гамма" (50% ООО "Альфа")

##### Акционеры

| # | Акционер | % |
|---|----------|---|
| 1 | Сидоров С.С. | 100% |
```

### Шаг 3. Расчёт effective ownership

```markdown
## Cascade analysis

### Иванов И.И.
- Direct в A: 30%
- **Effective ownership:** 30%
- → НЕ бенефициарный владелец (< 25%? Нет, 30% > 25%) → **БЕНЕФИЦИАР**

### Петров П.П.
- Через ООО "Альфа" 50%, которая владеет 60% в A
- Effective: 50% × 60% = **30%**
- → **БЕНЕФИЦИАР** (>25%)

### Сидоров С.С.
- Через АО "Гамма" 100% → ООО "Альфа" 50% → A 60%
- Effective: 100% × 50% × 60% = **30%**
- → **БЕНЕФИЦИАР**

### ООО "Бета" (10% юр.лицо A)
- Direct 10% < 25%, не бенефициар через ownership чейн
- НО — если есть control beyond ownership (например, по корп.договору) — отдельный анализ

## Бенефициары юр.лица A

1. Иванов И.И. — 30% direct
2. Петров П.П. — 30% через ООО "Альфа"
3. Сидоров С.С. — 30% через АО "Гамма" → ООО "Альфа"
```

### Шаг 4. Проверка контроля beyond ownership

Cases когда физ.лицо может быть БВ без прямого / косвенного 25%+ владения:

| Pattern | Possible БВ |
|---------|-------------|
| Trustee / номинальный держатель | Реальный бенефициар (через trust agreement) |
| Управляющая компания | Лицо контролирующее УК |
| Корпоративный договор с veto rights | Лицо с veto |
| Опционы / convertibles | Лицо с правом на conversion |
| Семейная structure | Глава семьи / patriarch |
| Trustee / директор-номинал | Реальный benefitee |

### Шаг 5. Документация

```markdown
## Бенефициарные владельцы [Юр.лицо A]

### БВ 1: Иванов Иван Иванович
- Дата рождения: [DD.MM.YYYY]
- Гражданство: РФ
- Документ: паспорт серия XX № YYYYYY, выдан [...]
- Адрес: [...]
- ИНН: [...]
- Способ обладания: прямое участие 30% в УК

### БВ 2: Петров Пётр Петрович
- ...
- Способ обладания: косвенное участие 30% через ООО "Альфа"

### БВ 3: ...

## Иные лица, контролирующие действия клиента

[Если есть — non-ownership control]

## Источник информации

- ЕГРЮЛ выписки на [DD.MM.YYYY]
- Учредительные документы (приложение)
- Корп.договор (если есть и accessible)
- Заявление клиента + paport данные БВ
- Дополнительные источники (СПАРК, Контур, открытые)
```

### Шаг 6. Action при выявлении

#### Apply sanctions / PEP screening к каждому БВ

→ см. `/sanctions-screening` + `/pep-screening`

#### Document для регуляторных целей

- В KYC dossier клиента
- В декларации о бенефициарах в ФНС (НК ст.93.3) — обновление при изменениях
- В заявлении в ФНС по 215-ФЗ public reestre

#### Periodic re-verification

- Annual minimum
- At любых corporate changes (новые участники, реорганизация)

## Red flags при анализе

### 🔴 Critical

- **Невозможно идентифицировать БВ** (опаковая структура с trust в low-transparency jurisdiction) → STOP, decline / EDD
- **БВ в sanctions list** → freeze
- **БВ — PEP без disclosure** → EDD
- **Шарка / номиналы** (явные nominees на уровне директоров) → reject / EDD

### 🟡 Medium

- БВ из недружественной страны → дополнительная проверка по Указам РФ
- Cascade > 3 уровней → enhanced documentation
- БВ refuses disclosure → подозрение → EDD

### 🟢 Cleared

- БВ — резидент РФ, прозрачный business
- Не в sanctions / PEP lists
- Documents complete и consistent

## Резервы (с 2025)

С 2025 году в РФ — растущие требования к раскрытию БВ через **публичный реестр** (215-ФЗ). Action:
- Ensure ПВК отражают актуальные требования
- Update procedures при новых изменениях

## Что НЕ делает

- Не handles изменения в ЕГРЮЛ
- Не подаёт декларации в ФНС (это бухгалтерия / юр.отдел)
- Не handles несоответствия с client при refusing disclosure — это compliance officer / outside-адв

**Правовая основа:**
- ФЗ-115 ст.3 (определение БВ)
- ФЗ-115 ст.7 (обязанность идентифицировать)
- ФЗ-208 ст.93.1 (для АО — раскрытие)
- НК ст.93.3 (декларация о бенефициарах в ФНС)
- ФЗ-215 (публичный реестр бенефициаров)
- Постановления Правительства / ЦБ (детализация)
