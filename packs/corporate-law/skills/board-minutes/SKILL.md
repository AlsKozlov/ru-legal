# /board-minutes

## Назначение

Draft протокол для:
- **Очередное / внеочередное Общее собрание участников ООО (ОСУ)**
- **Очередное / внеочередное Общее собрание акционеров АО (ОСА)**
- **Заседание Совета директоров (СД)**
- **Заседание Правления (для АО с коллегиальным исп.органом)**

(Для заочного голосования см. `/written-consent`.)

## Pre-flight

- **Тип собрания**: ОСУ / ОСА / СД / Правление
- **Тип организации**: ООО / АО (непубличное / ПАО)
- **Очередное / внеочередное**: для очередного — обязательные annual matters; для внеочередного — конкретная повестка
- **Повестка дня** определена
- **Кворум присутствующих** известен
- **Нужно ли нотариальное удостоверение** (для ООО ОСУ — yes по умолчанию)

## Mandatory contents (ст.181.2 ГК — для любого протокола)

Все протоколы должны содержать:

1. **Дата, время и место** проведения собрания
2. **Сведения о лицах, принявших участие**
3. **Результаты голосования** по каждому вопросу
4. **Сведения о лицах, голосовавших против** + потребовавших внести их позиции в протокол (если такие были)
5. **Сведения о лицах, проводивших подсчёт голосов**
6. **Сведения о лицах, удостоверивших протокол** (нотариус для ООО / регистратор для АО, или иные предусмотренные уставом лица)

**Несоблюдение** этих требований → **оспоримость** (ст.181.4 ГК).

## Workflow

### Шаг 1. Identify тип + повестка

```markdown
- Тип: [ОСУ ООО / ОСА АО непубличного / ОСА ПАО / СД]
- Характер: [очередное / внеочередное]
- Дата: [DD.MM.YYYY], время [HH:MM]
- Место: [адрес]
- Повестка дня:
  1. ...
  2. ...
```

### Шаг 2. Pre-flight checks

#### Срок уведомления участников / акционеров

- ООО (ст.36 ФЗ-14): не менее **30 дней** до даты
- АО непубличное (ст.52 ФЗ-208): не менее **21 дня**
- ПАО (ст.52 ФЗ-208): не менее **30 дней**
- СД: по уставу / регламенту СД — обычно 7-14 дней

**Если срок нарушен** → flag, решение оспоримо.

#### Кворум

- ООО ОСУ (ст.37 ФЗ-14): по уставу, по умолчанию 50% от голосов
- АО ОСА (ст.58 ФЗ-208): по типу вопроса — 50% обычные; 75% реорганизация / ликвидация / изменение УК; 100% некоторые
- СД (по уставу): обычно простое большинство присутствующих

### Шаг 3. Draft протокол

#### Template: ОСУ (ООО)

```markdown
# ПРОТОКОЛ № [N]
# [очередного / внеочередного] общего собрания участников
# Общества с ограниченной ответственностью "[Наименование]"

**ОГРН:** [номер]
**Место проведения:** [адрес]
**Дата проведения:** [DD.MM.YYYY]
**Время начала:** [HH:MM]
**Время окончания:** [HH:MM]
**Форма проведения:** очное собрание

## Сведения о председательствующем и секретаре

- **Председательствующий:** ФИО, должность / роль
- **Секретарь:** ФИО

## Сведения об участниках

| # | ФИО / наименование участника | Доля в УК, % | Присутствует лично / по доверенности |
|---|------------------------------|--------------|--------------------------------------|
| 1 | Иванов И.И. | 30% | лично |
| 2 | ООО "Альфа" (ИНН XXX) | 40% | представитель Сидоров С.С. (доверенность № X от DD.MM.YYYY) |
| 3 | Петров П.П. | 30% | не присутствует |

**Всего:** [N] участников с общей долей [X]% от УК

## Кворум

Согласно ст.37 ФЗ-14 / Уставу общества п.[X] кворум для проведения общего
собрания составляет [X]% голосов.

Присутствует [Y]% — кворум **[имеется / отсутствует]**.

## Повестка дня

1. [Вопрос 1]
2. [Вопрос 2]
3. ...

## Решения по повестке дня

### Вопрос 1: [формулировка]

**Слушали:** [ФИО — кто докладывал, краткое содержание выступления]

**Выступили:** [ФИО — кто выступил с вопросами / возражениями]

**Голосовали:**
- За: [N голосов, X%]
- Против: [M голосов, Y%]
- Воздержались: [K голосов, Z%]

**Решение** [принято / не принято]

**[Если принято — текст решения дословно:]**

> [Текст решения]

### Вопрос 2: [...]

[То же]

## Удостоверение протокола

Настоящий протокол [нотариально удостоверён нотариусом ФИО, реестр N, дата] /
[удостоверён в порядке, предусмотренном Уставом п.X — описание способа]
[/ подписан всеми участниками (если предусмотрено уставом)].

## Подписи

- Председательствующий: ________________ / [ФИО] /
- Секретарь: ________________ / [ФИО] /

**Дата составления протокола:** [DD.MM.YYYY]
```

- Для ПАО — добавить сведения для раскрытия (number акций каждого типа, представители)

### Шаг 4. Pre-publication checks

- [ ] Все обязательные content по ст.181.2 включены
- [ ] Кворум documented + reference к норме / уставу
- [ ] Каждый вопрос: слушали → выступили → голосовали → решение
- [ ] Тексты решений сформулированы **дословно как принято** (для последующего исполнения)
- [ ] Для ООО — план нотариального удостоверения (нотариус booked? альтернативный способ предусмотрен уставом?)
- [ ] Для АО — план удостоверения через регистратора
- [ ] Если решение влечёт изменения в ЕГРЮЛ — план подачи Р13014 в 7 раб.дней

### Шаг 5. Risk flags

#### 🔴 Critical

- **Срок уведомления нарушен** → решение оспоримо (ст.43 ФЗ-14 / ст.49 ФЗ-208)
- **Кворум фактически отсутствовал** → решение **ничтожно** (ст.181.5 ГК)
- **Решение по вопросу, не включённому в повестку** → ничтожно (ст.181.5 ГК)
- **Принято решение по вопросу вне компетенции** → ничтожно
- **Нотариальное удостоверение для ООО не выполнено + уставом не предусмотрен альтернативный способ** → оспоримо (ст.181.4)

#### 🟡 Medium

- Документы / материалы по повестке не были предоставлены за установленный срок до собрания
- Технические недостатки протокола (опечатки в долях, suboptimal формулировки решений)
- Отсутствие mentions выступлений / особых мнений (если они были)

#### 🟢 Minor / informational

- Оформление по форме (шрифт, поля, и т.п.)

## Что НЕ делает

- Не proводит собрание (это администрирование)
- Не удостоверяет нотариально (нужен живой нотариус)
- Не подписывает за председателя / секретаря
- Не подает форму Р13014 в ФНС
