# /corporate-meeting-prep

## Назначение

Подготовка к **очному** общему собранию / заседанию СД. Для заочного — `/written-consent`.
Для оформления протокола post-meeting — `/board-minutes`.

Этот skill = pre-meeting planning + draft материалов.

## Pre-flight

- **Тип собрания:**
  - Очередное / годовое ОСУ или ОСА
  - Внеочередное ОСУ / ОСА
  - Заседание СД
  - Заседание Правления
- **Инициатор:** [ГД / участник / акционер ≥ 10% / СД]
- **Тип организации:** ООО / АО / ПАО
- **Повестка дня — initial draft**

## Workflow

### Шаг 1. Define повестку дня + check компетенцию

#### Для ООО ОСУ — компетенция (ст.33 ФЗ-14)

**Исключительная компетенция:**
- Изменение устава, размера УК
- Назначение / прекращение полномочий ГД (если устав не передаёт СД)
- Утверждение годовой бух.отчётности
- Распределение прибыли
- Реорганизация / ликвидация
- Образование исполнительных органов / назначение СД / ревизионной комиссии
- Утверждение аудитора
- Иное

#### Для АО ОСА — компетенция (ст.48 ФЗ-208)

Аналогично, more detailed:
- Изменение устава, размера УК (увеличение / уменьшение)
- Реорганизация, ликвидация
- Избрание СД и ревизионной комиссии (досрочное прекращение их полномочий)
- Назначение / досрочное прекращение полномочий единоличного и членов коллегиального исп.органов (если не передано СД)
- Утверждение аудитора
- Утверждение годового отчёта, годовой бух.отчётности, распределения прибыли
- Решение об увеличении / уменьшении УК
- Утверждение крупных сделок (если ≥ 50% активов)
- Иное

#### Для СД — компетенция

По уставу. Common items:
- Определение приоритетных направлений деятельности
- Созыв ОСУ / ОСА
- Утверждение повестки дня ОСУ / ОСА
- Назначение / прекращение полномочий ГД (если устав передал)
- Согласование крупных сделок (от 25% до 50% активов — если устав)
- Согласование сделок с заинтересованностью
- Утверждение внутренних документов

**Check:** все ли вопросы повестки в компетенции данного органа? Если **нет** — решение **ничтожно** (ст.181.5 ГК).

### Шаг 2. Determine cрок уведомления

| Орган | Срок до даты |
|-------|--------------|
| ООО ОСУ | **не менее 30 дней** (ст.36 ФЗ-14) |
| АО непубличное ОСА | **не менее 21 дня** (ст.52 ФЗ-208) |
| ПАО ОСА | **не менее 30 дней** (ст.52 ФЗ-208) |
| Внеочередное ОСУ ООО — для крупных сделок / связанных с заинтересованностью | **не менее 30 дней** |
| ОСА АО — если повестка включает реорганизацию или избрание СД | **не менее 50 дней** |
| СД | по уставу — typical 7-14 дней |

**Если срок нарушен** → решения оспоримы (ст.181.4 ГК).

### Шаг 3. Determine кворум

| Орган | Кворум |
|-------|--------|
| ООО ОСУ | По уставу, по умолчанию 50% от голосов (ст.37 ФЗ-14) |
| АО ОСА — обычные вопросы | **50%** + 1 (ст.58 ФЗ-208) |
| АО ОСА — реорганизация / ликвидация / изменение устава в части прав акционеров | **75%** |
| АО ОСА — общее решение об увеличении УК через размещение | По уставу, обычно **75%** |
| АО ОСА — повторное собрание (если первое не имело кворума) | **30%** (ст.58 ФЗ-208 ч.3) |
| СД | По уставу — typical 50%+1 |

**Если на собрании кворум фактически отсутствует** → решения **ничтожны** (ст.181.5 ГК).

### Шаг 4. Draft уведомление

```markdown
# Уведомление о проведении [очередного / внеочередного / годового]
# общего собрания участников / акционеров / заседания СД

**Кому:** [участник / акционер / член СД — ФИО / наименование]

[Организационно-правовая форма + наименование] уведомляет о проведении [тип собрания] в соответствии с [ст.36 ФЗ-14 / ст.52 ФЗ-208 / Уставом п.X].

## Дата, время, место

- **Дата:** [DD.MM.YYYY]
- **Время начала регистрации:** [HH:MM]
- **Время открытия собрания:** [HH:MM]
- **Место:** [полный адрес]
- **Форма:** [очное собрание / совместное присутствие]

## Повестка дня

1. [Вопрос 1] — [краткая формулировка]
2. [Вопрос 2]
3. ...

## Регистрация

Регистрация участников проводится с [HH:MM] до [HH:MM] по адресу [...].
При себе иметь:
- Паспорт (для физ.лица)
- Доверенность (для представителя) — нотариально удостоверённую
- Учредительные документы (для юр.лица-участника)

## Материалы для ознакомления

С материалами по повестке дня можно ознакомиться:
- По адресу [...] с [время] по [время] в рабочие дни с [дата по дата]
- Электронно по запросу на email [...]

## Прилагаемые документы

[List что прикладывается к уведомлению]

С уважением,
[ФИО, должность подписавшего]
[Подпись]
[Дата]
```

### Шаг 5. Prepare материалы по повестке дня

Для каждого вопроса:

#### Документы которые должны быть готовы

- **Утверждение бух.отчётности** — годовая отчётность + аудиторское заключение
- **Распределение прибыли** — проект решения с конкретными цифрами
- **Назначение ГД** — кандидатура (ФИО + биография + предложение по сроку полномочий)
- **Крупная сделка** — текст сделки + сведения о цене / стоимости активов
- **Сделка с заинтересованностью** — раскрытие заинтересованности, цена, условия
- **Изменение устава** — текст изменений + сравнение прежней и новой редакции
- **Иное** — соответствующие документы

#### Бюллетень для голосования (опционально, обычно для АО)

См. template в `/written-consent`.

### Шаг 6. Логистика

- Зал / помещение booked
- Регистратор присутствует (для АО) — coordination с регистратором
- Нотариус booked (для ООО, если требуется удостоверение по ст.67.1 ГК)
- Технические средства (камеры, проектор, эл.голосование если planned)
- Запись (аудио / видео если предусмотрено уставом — допустимо)
- Кейтеринг (для long meetings)

### Шаг 7. Pre-meeting checklist

```markdown
# Pre-Meeting Checklist — [тип собрания, дата]

## За 60+ дней до даты

- [ ] Решение о созыве принято (инициатор: ГД / участник ≥ 10% / СД)
- [ ] Повестка дня определена с проверкой компетенции
- [ ] Срок уведомления планировать (минимум 30 / 21 / 50 дней)

## За 30 дней до даты

- [ ] Уведомления отправлены всем участникам / акционерам
- [ ] Способ направления: вручение под расписку / заказное письмо / email (если устав предусматривает)
- [ ] Doc подтверждающие отправку сохранены (квитанции, email logs)

## За 21 день до даты (для АО непубличного)

- [ ] Список лиц, имеющих право участвовать — сформирован (ст.51 ФЗ-208)
- [ ] Регистратор уведомлён

## За 7-10 дней до даты

- [ ] Материалы для ознакомления готовы и доступны
- [ ] Зал / технические средства booked
- [ ] Нотариус booked (для ООО)
- [ ] Регистратор подтвердил участие (для АО)

## За 1 день до даты

- [ ] Печатные copies материалов и бюллетеней (если используются) готовы
- [ ] Регистрационный лист подготовлен
- [ ] Председательствующий + секретарь briefed
- [ ] Контактные данные участников для late RSVPs

## День собрания

- [ ] Регистрация — все участники verified (паспорта, доверенности)
- [ ] Quorum определён до начала
- [ ] Председательствующий объявляет открытие
- [ ] Поэтапное прохождение повестки
- [ ] Подсчёт голосов по каждому вопросу
- [ ] Нотариус удостоверяет (для ООО)
- [ ] Закрытие собрания

## После собрания

- [ ] Протокол draft — см. `/board-minutes`
- [ ] Удостоверение протокола (нотариус для ООО)
- [ ] Подача документов в ФНС (Р13014) в 7 раб.дней если решения требуют изменения ЕГРЮЛ
- [ ] Распространение протокола (внутреннее + регистраторской АО)
- [ ] Действия по принятым решениям (приказы, исполнение)
```

### Шаг 8. Common gotchas

#### 🔴 Critical

- **Вопрос вне компетенции** этого органа → решение ничтожно
- **Срок уведомления нарушен** → решения оспоримы
- **Нет кворума** → собрание не состоявшееся; решения ничтожны
- **Нотариус не присутствует** на ООО ОСУ (или альтернативный способ удостоверения по уставу не выполнен) → решения оспоримы

#### 🟡 Medium

- Материалы по повестке предоставлены слишком поздно (нарушение ст.36 ч.3 ФЗ-14 — за 30 дней)
- Доверенности участников оформлены неправильно (не нотариально удостоверены)
- Решение по вопросу, не точно сформулированному в повестке (deviation)

#### 🟢 Acceptable

- Минор issues по форме регистрации (но не подрывающие validity)
- Опечатки в материалах (исправляются на месте)

## Что НЕ делает

- Не проводит собрание (это администрирование)
- Не draft протокол post-meeting (это `/board-minutes`)
- Не подаёт в ФНС после изменений (это секретарь / outside-адв)

## Why RU-unique

В US:
- Notice requirements typically shorter (часто 10 дней по Delaware GCL §222)
- Quorum по default = простое большинство (1/3 или ½ по уставу)
- Нет нотариального удостоверения корп.решений (US LLC / corp resolutions не требуют notarization)
- Compétence органов более flexible (operating agreement / bylaws определяют)
- Нет регистратора-third-party для maintain reestre

→ РФ требует strictly structured pre-meeting workflow с specific deadlines + actors.

**Правовая основа:**
- ФЗ-14 «Об ООО» (ст.33, 36-38)
- ФЗ-208 «Об АО» (ст.48, 51-52, 58)
- ГК (ст.181.2, 181.4, 181.5)
- ст.67.1 ГК — нотариальное удостоверение
- ФЗ-39 ст.8 — регистратор АО
