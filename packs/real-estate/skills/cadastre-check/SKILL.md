# /cadastre-check

## Назначение

Verify property через ЕГРН — official Российский registry недвижимости. First
step DD для any real estate transaction.

## Pre-flight

- **Кадастровый номер** ИЛИ адрес ИЛИ владелец data
- **Cost expectation:** платная выписка обычно ~290-460 руб через Росреестр
- **Timeline:** electronic — minutes; paper — 3-5 раб.дней

## What can быть verified through ЕГРН

### A. Объектная информация

- Точный адрес
- Площадь (общая, жилая для квартир)
- Этажность, год постройки
- Назначение (жилое / нежилое / индустриальное / иное)
- Кадастровая стоимость

### B. Права

- Текущий собственник (физ.лицо / ЮЛ)
- Дата + основание возникновения права
- Доли (если несколько собственников)
- История переходов (последние сделки)

### C. Обременения

- Ипотека (с указанием банка)
- Аренда (зарегистрированная)
- Сервитут
- Арест / запрет регистрационных действий
- Доверительное управление
- Иное

### D. Кадастровая стоимость

- Текущая кадастровая стоимость (база для налога на имущество)
- Дата определения
- Возможность оспаривания через комиссию Росреестра / суд

## Workflow

### Шаг 1. Get выписку из ЕГРН

#### Channels

1. **Сайт Росреестра** (rosreestr.gov.ru) — preferred
2. **Госуслуги** — for физ.лиц проще
3. **МФЦ** — paper / выписка с печатью

#### Types выписок

- **Об основных характеристиках + правах** — общая (typical)
- **О переходе прав** — history переходов
- **О кадастровой стоимости** — для оспаривания
- **О полной информации** — все категории (paid version)
- **Об обременениях** — focused
- **Расширенная** — special purposes

### Шаг 2. Analyze object data

```markdown
## Объектная analysis

- **Кадастровый номер:** [...]
- **Адрес:** [...]
- **Точное соответствие** с указанным в сделке: [Y/N]
- **Назначение:** [жилое / нежилое / иное]
- **Площадь:** общая [X м²] / жилая [Y м²]
- **Этаж / этажность:** [...]
- **Год постройки:** [...]
- **Кадастровая стоимость:** [сумма]

## Соответствие планируемой сделке

- [ ] Площадь match
- [ ] Назначение match (если жилое для жилой — OK; нежилое — apartments? watch!)
- [ ] Адрес match
```

### Шаг 3. Verify rights

```markdown
## Права analysis

- **Текущий собственник(и):** [ФИО / ЮЛ]
- **Доли:** [если несколько]
- **Дата приобретения:** [DD.MM.YYYY]
- **Основание (договор, наследство, приватизация, и т.д.):** [...]

### Match с продавцом / стороной сделки

- [ ] Имя match exactly
- [ ] Все собственники identified
- [ ] Если в браке — согласие супруга нужно для отчуждения

### History переходов (если получили эту выписку)

- Last few transactions — pattern normal? Suspicious quick flips?
- Banking / залоговые операции
- Часть сделок < 3 лет назад → potentially still подлежащие оспариванию по налогу при продаже (для физ.лиц) / банкротству (если применимо)
```

### Шаг 4. Verify обременения

```markdown
## Обременения analysis

### Ипотека

- **Есть?** [Y/N]
- **Банк-залогодержатель:** [...]
- **Когда зарегистрирована:** [...]
- **Implications для сделки:**
  - Если покупка — нужен consent банка-залогодержателя ИЛИ полное погашение
  - Если аренда — banking может contest

### Аренда (зарегистрированная)

- **Есть?** [Y/N]
- **Срок:** [...]
- **Арендатор:** [...]
- **Implications:**
  - При продаже — покупатель получает квартиру с этим арендатором (unless lease terminates)
  - Преимущественное право продления (ст.621 ГК)

### Сервитут

- **Есть?** [Y/N — usually для земельных участков]
- **В чьих интересах:** [...]
- **Implications:** ограничивает use объекта

### Арест / запрет

- **Есть?** [Y/N — CRITICAL]
- **Кто наложил:** [ФССП / суд / иное]
- **Основание:** [...]
- **Implications:** **сделка невозможна** до снятия

### Доверительное управление

- **Есть?** [Y/N]
- **Управляющий:** [...]

⚠ **Critical: если есть обременения** — мust be resolved before или provisions добавлены в договор для resolving.
```

### Шаг 5. Verify cadastre стоимость

```markdown
## Кадастровая стоимость analysis

- **Текущая:** [сумма]
- **Дата определения:** [DD.MM.YYYY]

### Implications

- **Налог на имущество** — based on this value
- **НДФЛ при продаже** (для физ.лиц) — если рыночная > кадастровой × 0.7 — uses фактическая; если меньше — кадастровая × 0.7 (НК ст.214.10)
- **Госпошлина за регистрацию** — % от cadastral value

### Если оценка завышена

- Может быть оспорена через комиссию Росреестра / суд
- Снижение → reduce property tax + may benefit при продаже
- Reference: ст.22 ФЗ-237 «О государственной кадастровой оценке»
```

### Шаг 6. Цвет risks

```markdown
## Red flags

### 🔴 Critical (STOP)

- **Арест / запрет** на регистрационные действия
- **Несоответствие** собственника заявленному продавцу
- **Объект** в реестре проблемных застройщиков
- **Кадастровая стоимость 0** или missing — возможны проблемы с regstrationом

### 🟡 Material

- **Ипотека** без согласия банка
- **Аренда** unknown к продавцу
- **Quick flips** в истории
- **Несоответствие** площади / иных характеристик

### 🟢 OK

- Чистая history
- Один stable собственник долгое время
- Нет обременений
- Cadastre data consistent
```

## Дополнительные services через ЕГРН

### A. Уведомления о изменениях

- Можно set up уведомления о изменениях в record объекта
- Через Госуслуги или личный кабинет Росреестра
- Useful для long-term monitoring

### B. Выписки о ваших правах

- Получить summary всех ваших объектов недвижимости в РФ
- Helpful для personal portfolio management

## Что НЕ делает

- Не handles формальную DD beyond cadastre (см. `/encumbrance-analysis` отдельно)
- Не registers сделку (это нотариус / Росреестр)
- Не оспаривает кадастровую стоимость (это outside-адв + Росреестр комиссия / суд)
