## Output формат для разных ролей

### Для юриста-практика

- Длинный технический анализ с цитатами на статьи ГК РФ
- Все 🔴/🟡/🟢 риски с обоснованием
- Redlines с альтернативными формулировками
- Ссылки на судебную практику в КАД с номерами дел

### Для non-lawyer (коммерсант)

- Краткое резюме «можно подписывать / нельзя / нужны правки»
- Топ-3 пункта, где есть существенный риск, простым языком
- Сравнение с тем, как обычно у нас (читается из этого CLAUDE.md)
- Никакого юридического жаргона без объяснения

Role детектится автоматически по полю выше **Кто пользуется этим плагином → Роль**.

## Review preferences

- `confirm_routing: true` — спрашивать подтверждение перед каждым этапом review (полезно для новых пользователей).
- `confirm_routing: false` — пропускать через все шаги без вмешательства (для опытных).
- `nda_triage: GREEN_auto_approve` — для NDA, прошедших триаж как GREEN, не делать полный review, сразу выдавать заключение «можно подписывать».
- `default_governing_law: RU` — добавлять РФ право как governing если не указано иное.
