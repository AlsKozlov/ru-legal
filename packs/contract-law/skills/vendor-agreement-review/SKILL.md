# Vendor Agreement Review (РФ adaptation)

## Назначение

Читает vendor agreement против playbook этой команды (PROFILE.md), находит каждый
term который отклоняется, и говорит юристу что с ним делать — с **конкретным**
redline'ом, не vague "consider revising".

---

## РФ critical context (читай **перед** review)

В отличие от US MSA paradigm, в РФ договорах с поставщиками **критичны 3 момента**:

### 1. Существенные условия = критично (ст.432 ГК)

Без них договор **незаключён** — взыскивать в суде сложно, нет договорной неустойки.

| Тип договора | Существенные условия | Норма |
|--------------|---------------------|-------|
| Услуги (гл.39) | Предмет (конкретные действия) | ст.779 ГК |
| Подряд (гл.37) | Предмет + **начальный и конечный сроки** | ст.702, ст.708 ГК |
| Поставка (§3 гл.30) | Наименование товара + количество | ст.455 ГК |

**Pitfall:** "оказание юридических услуг" без конкретизации = незаключённый договор
(ПП ВС РФ №49 от 25.12.2018, п.43).

### 2. Limit of Liability — рабочий, но с ловушками

- **Работает** для non-intentional breach (принцип свободы договора ст.421 + ст.400)
- **НЕ работает** при умысле — ст.401 п.4 ГК: ограничение ответственности за **умысел**
  ничтожно. LoL обнуляется.
- **Не работает** если cap ниже 2x ключевой ставки ЦБ за период просрочки (суды
  могут не применить ограничение).

### 3. Indemnity = двойной механизм (ст.431.2 + ст.406.1)

В РФ нет точного аналога US indemnification. Есть **два** отдельных инструмента:

| Институт | Когда применять | Особенности |
|----------|-----------------|-------------|
| **Ст.431.2 (заверения об обстоятельствах)** | Insurance подход — сторона заверяет о фактах (нет налоговых задолженностей, чистота прав, нет sanctions). При недостоверности — убытки. | Объективная ответственность, не требует вины. ПП ВС РФ №49 от 25.12.2018. |
| **Ст.406.1 (возмещение потерь)** | Allocation подход — компенсация при наступлении оговорённых обстоятельств (изменение налогов, санкции, претензии регуляторов). Без доказательства нарушения. | Только в предпринимательских договорах. |

**Critical:** нельзя применить **обе** статьи к одному риску без оговорки — суд
присудит только по одному основанию.

## Pre-flight

1. **Загрузи PROFILE.md** — если placeholder'ы → stop, попроси cold-start
2. **Определи сторону:** sell-side или purchase-side?
3. **Определи тип договора** (см. routing в `/review`):
   - Услуги (гл.39 ГК)
   - Подряд (гл.37 ГК)
   - Поставка (§3 гл.30 ГК)
   - Смешанный (ст.421 п.3)

Для **смешанных** договоров — применяй нормы каждой части. Не пытайся унифицировать.

## Workflow

### Шаг 1. Проверь существенные условия (gate)

Это **первый check** — без него остальной review бесполезен.

#### Для услуг (гл.39 ГК)
- [ ] Предмет: **конкретные действия** исполнителя описаны?
- [ ] Стороны явно указаны?

#### Для подряда (гл.37 ГК)
- [ ] Предмет: вид и **объём работ**?
- [ ] **Начальный срок** работ указан?
- [ ] **Конечный срок** работ указан?

#### Для поставки (§3 гл.30 ГК)
- [ ] Наименование товара конкретное?
- [ ] Количество определено?

**Если любое условие отсутствует** — auto-RED:

> "Существенное условие для [тип договора] отсутствует: [конкретно что]. По ст.432
> ГК + ПП ВС РФ №49 (п.43) это влечёт **незаключённость** договора. Без него:
> - Договорная неустойка не работает
> - Гарантийные сроки (ст.722 для подряда) не применяются
> - Сложнее взыскать убытки — придётся через неосновательное обогащение
>
> Рекомендация: до подписания требовать конкретизацию [предмета / сроков / наименования]."

### Шаг 2. Параграф-by-параграф review с playbook

Для каждого пункта договора:

1. **Identify clause type** (см. таблицу типов в `contract-review` skill)
2. **Compare к playbook** в PROFILE.md → `## Playbook` → matching side
3. **Check РФ норму** (см. таблицу проверок ниже)
4. **Classify risk** 🔴/🟡/🟢
5. **Suggest surgical redline** (smallest possible — слово > фраза > предложение)

### Шаг 3. Cross-cutting compliance checks

См. секции "Compliance" ниже.

### Шаг 4. Эскалации

Если issue превышает authority текущего reviewer'а → invoke `escalation-flagger`.

### Шаг 5. Output

Полный memo (см. format ниже).

## Категории проверок (РФ-specific)

### 1. Ответственность и LoL

**Проверки:**
- LoL cap определён?
- Покрывает ли LoL **только** non-intentional breach?
- **Auto-RED:** если LoL пытается ограничить ответственность за умысел (ст.401 п.4 — ничтожно)
- Соотносится ли LoL с неустойкой? (нужна оговорка об **исключительной** неустойке по
  ст.394, иначе убытки взыскиваются сверх неустойки по умолчанию)

**Recommended GREEN формулировки:**

> "Стороны соглашаются, что общий размер ответственности Исполнителя по настоящему
> Договору не может превышать [сумму], за исключением случаев умышленного нарушения
> и грубой неосторожности."

Это явная оговорка про ст.401 п.4 — упрощает enforcement.

### 2. Неустойка

**Проверки:**
- Размер? (>0.5% в день = YELLOW — будет снижена по ст.333)
- Оговорка про **исключительную** vs **зачётную** неустойку (ст.394)?
- Включена ли неустойка в общий LoL cap?

**Pitfall:** "штраф 50% от стоимости договора за разглашение конфиденциальной
информации" — суд снизит по ст.333 до 5-15% типично.

### 3. Заверения (ст.431.2)

**Типичные заверения для vendor:**
- О действительности учредительных документов
- О полномочиях подписанта
- О отсутствии санкций (для иностранных контрагентов)
- О действительности лицензий (для регулируемой деятельности)
- О чистоте прав на передаваемые результаты работ (IP)

**Recommended:** Включить **explicit** заверения с **explicit** последствиями
(убытки или неустойка) — это даёт sophisticated РФ инструмент защиты с объективной
ответственностью.

**Если заверений нет** — **YELLOW** (упущена защита).

### 4. Возмещение потерь (ст.406.1)

**Когда применять:**
- Изменение налогового законодательства влияет на цену
- Sanctions block платежей
- Регуляторные претензии (РКН по 152-ФЗ, ФАС, и т.д.)

**Critical:** если в договоре уже есть ст.431.2 заверения по тому же риску — нужна
оговорка о запрете двойного взыскания. Иначе суд присудит только одно.

### 5. Гарантии (для подряда — ст.722-725 ГК)

**Проверки:**
- Гарантийный срок указан? (если нет — default ст.722)
- Гарантийный срок ≥ срок службы? (ст.722 — гарантийный должен покрывать
  предполагаемый срок службы)
- Что включает гарантия — исправление, замена, возмещение?

### 6. Расторжение

**Ст.450.1 ГК — отказ от договора:**
- Условия одностороннего отказа explicit?
- Срок уведомления указан?
- Возмещение расходов при отказе урегулировано?

**Определение ВС РФ №310-ЭС25-10 (04.03.2025):** Заказчик имеет право на
**односторонний отказ** по ст.450.1 при ненадлежащем исполнении без ожидания
судебного расторжения. Договорная оговорка действительна для B2B.

### 7. Подсудность и применимое право

**🟢 GREEN:**
- Применимое право: РФ
- Подсудность: АС по месту ответчика (ст.27-32 АПК)
- Альтернатива для крупных: МКАС при ТПП РФ

**🟡 YELLOW:**
- Иностранное право для B2B — РФ суды применят свои imperative norms
- LCIA / ICC — текущие restrictions для подсанкционных лиц

**🔴 RED:**
- Иностранное право + подсудность вне РФ — может быть unenforceable
- Арбитраж в инstituтах которые приостановили дела для РФ лиц

### 8. Force Majeure (ст.401 ГК)

**Проверки:**
- Перечень обстоятельств указан конкретно?
- Срок уведомления о наступлении?
- Документальное подтверждение (заключение ТПП)?

**РФ practice:**
- **МТПП с 04.10.2023 приостановила выдачу** заключений по внутрироссийским
  договорам в связи с санкциями
- Санкции **сами по себе ≠ форс-мажор** для договоров, заключённых после 02.2022
  (предсказуемость)
- "Экономические трудности" — суды **не признают** как форс-мажор

### 9. IP / Интеллектуальная собственность

**Critical для разработки ПО / контентa:**

- **Ст.1295 ГК (служебное произведение):** права автоматически у работодателя vendor'а,
  не у заказчика. **Work-for-hire не существует в РФ.**
- Для передачи прав заказчику нужен **отдельный**:
  - Договор отчуждения (ст.1234) — полная передача исключительных прав
  - Или лицензионный договор (ст.1235) — лицензия (исключительная или неисключительная)

**🔴 Auto-RED:** "исполнитель передаёт все права" без указания вида (отчуждение vs
лицензия) — недостаточно. ВС требует прямого указания.

**Recommended GREEN формулировка для разработки:**

> "Исполнитель отчуждает Заказчику исключительное право на результаты работ (далее —
> Результат) в полном объёме на весь срок действия исключительного права на
> Результат и для всех способов использования, перечисленных в ст.1270 ГК РФ.
> Передача оформляется актом приёма-передачи."

### 10. Защита персональных данных (152-ФЗ)

**Если vendor обрабатывает ПДн заказчика:**

- [ ] Договор-поручение на обработку ПДн (ДОУ) есть? (ст.6 ч.3 152-ФЗ)
- [ ] Перечень ПДн, цель, действия указаны?
- [ ] Конфиденциальность обеспечена (ст.7)?
- [ ] Требования ст.18 и 18.1 (локализация) учтены?
- [ ] Обязанность безопасности (ст.19) возложена на vendor?
- [ ] Cross-border ПДн — уведомление РКН (ст.12)?

**С 30.05.2025 (ФЗ-420):** штрафы значительно увеличены, оператор несёт
ответственность за обработчика.

### 11. Налоговые риски

**Проверки:**
- **НДС:** "с НДС / без НДС / УСН" указано **явно**? Vendor на УСН → заказчик теряет
  входящий НДС.
- **Vendor-нерезидент:** заказчик = **налоговый агент** (ст.161 НК — НДС, ст.310 НК
  — налог на прибыль).
- **ТЦО (с 2024):** контролируемые все сделки с компаниями из стран с налогом <15%.
- **Дробление с ИП:** vendor — ИП с признаками трудовых отношений? Доначисление НДФЛ
  + взносы + штраф 40%.

### 12. Sanctions compliance

**Если vendor — иностранное лицо или РФ компания с иностранным контролем:**

- [ ] Контр-санкции (Указ Президента №79) учтены?
- [ ] Sanctions clause включена (право не проводить платёж + расторжение)?
- [ ] Платежи иностранцам — разрешение ПКО / Правительственной комиссии?
- [ ] Проверка контрагента по sanctions lists (manual в Phase 1, automated через
  sanctions-mcp в Phase 2)?

### 13. Коммерческая тайна / Конфиденциальность

См. секцию в `nda-review` skill про режим КТ по 98-ФЗ ст.10.

**Если конфиденциальность — отдельный пункт договора (не отдельный NDA):**
- Перечень КИ конкретный?
- Срок survival после прекращения договора?
- Carve-outs стандартные?

**Если защищаемая информация — реально критична** → рекомендация выделить отдельный
NDA с настоящим режимом КТ.

## Реальные кейсы из РФ practice

### А33-28614/2024 (АС Красноярска, 19.02.2025)

Неустойка по услугам — ст.333 ГК **снижена с 10% до 5%**. Заявление о снижении
должно быть мотивированным.

### Определение ВС РФ №310-ЭС25-10 (04.03.2025)

Право на односторонний отказ от услуг по ст.450.1 ГК — без судебного расторжения.

### Массовая практика АС МО / АС ПО — незаключённый подряд

Отсутствие сроков (ст.708) = незаключённый. Исполнитель → неосновательное обогащение,
теряет договорную неустойку.

### IP-кейсы АС г.Москвы (2023-2024)

Заказчик оплатил ПО, нет отчуждения по ст.1234 → vendor отказал в передаче кода.
Заказчик получает только лицензию или ничего.

### Санкционный форс-мажор (АС МО / АС СПб 2022-2023)

Массовые отказы — предсказуемость санкций для договоров после 02.2022.

### Переквалификация ИП → трудовые (ФНС 2023-2025)

Признаки: регулярность, единственный заказчик, нет инфраструктуры. Доначисления +
штраф 40%.

## Контрагент context

### BigCo / Госкорпорации

Часто настаивают на standard формах. Negotiate на старте, потом не изменишь.

### Госконтракты (ФЗ-44/223)

Specific procedures (см. отдельный `tender-contract-review` skill в `public-procurement`
pack, если установлен).

### Иностранные из недружественных стран

Auto-flag sanctions, валютный контроль, governing law restrictions.

### Стартапы

Часто примут вашу форму. Самый быстрый path — "используем нашу".

## Что этот skill НЕ делает

- Не negotiate'ит. Surface'ит issues и предлагает redlines.
- Не подписывает.
- Не принимает risk decisions сам — flag'ает.
- Не проверяет sanctions automatically (manual в Phase 1).
- Не валидирует подписи / полномочия — это compliance check вне scope.
