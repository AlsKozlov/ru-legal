# /subpoena-triage (запросы суда / органов)

## Назначение

В РФ "subpoena" — широкое понятие. Различные виды:

| Тип запроса | Норма | Forum | Срок ответа |
|-------------|-------|-------|-------------|
| **Запрос арбитражного суда** | АПК ст.66 (истребование доказательств) | АС | Указан в определении (обычно 5-10 дней) |
| **Запрос по поручению суда** | АПК ст.74 | АС | Указан |
| **Запрос судьи (СОЮ)** | ГПК ст.57 | СОЮ | Указан |
| **Запрос следователя (уголовный)** | УПК ст.183 (выемка), ст.182 (обыск) | Следствие | Незамедлительно |
| **Запрос прокуратуры** | ФЗ "О прокуратуре" ст.6, ст.22 | Прокуратура | Обычно 7-30 дней (указан) |
| **Запрос ФАС** | ФЗ-135 ст.25 | ФАС | Указан (обычно 7-15 дней) |
| **Запрос РКН** | 294-ФЗ + 152-ФЗ ст.23 | РКН | Указан (обычно 5-10 дней) |
| **Запрос ФНС** | НК ст.93, ст.93.1 (по поручению другой ФНС) | ФНС | 10 дней (ст.93 ч.3) |
| **CID Минфин (валютный контроль)** | 173-ФЗ ст.23 | Минфин | Указан |

Каждый имеет свою процедуру + санкции за неисполнение.

## ⚠️ Pre-flight critical

**Не игнорировать!** В РФ неисполнение запроса = штраф + обвинение в препятствовании
правосудию / производству (УК ст.296, ст.297 — некоторые случаи).

## Workflow

### Шаг 1. Identify тип запроса

Спроси если неясно:

- Кто отправитель? (Суд / следователь / прокурор / РКН / ФАС / ФНС)
- Какая ссылка на норму? (АПК ст.66, УПК ст.183, etc.)
- На каком этапе? (До иска / в рамках иска / уголовное дело / административное)

### Шаг 2. Classify по нормативной базе

**Запрос арбитражного суда (АПК ст.66):**
- Суд истребует документы по ходатайству стороны
- Мы — третье лицо, не сторона дела (если бы были стороной — другая процедура)
- Срок указан в определении

**Запрос следователя (УПК):**
- Письменное требование о выемке (ст.183)
- Или постановление об обыске (ст.182) — более серьёзная процедура
- **Адвокатская тайна** (если адвокат ФПА) — может быть основанием отказа

**Запрос прокуратуры:**
- В рамках общего надзора (ФЗ "О прокуратуре" ст.22) — широкие polномочия
- Сложно отклонить
- Могут запросить любые документы относящиеся к "соблюдению законов"

**Запрос регулятора (ФАС / РКН / ФНС):**
- В рамках надзорной функции
- 294-ФЗ ограничивает рамки (для inspections)
- 248-ФЗ — Inspections новый порядок

### Шаг 3. Scope analysis

Для каждого требования — оценить:

- **Конкретность:** запрос конкретный или fishing expedition?
- **Релевантность:** документы реально relevant для предмета запроса?
- **Burden:** объём — реально ли в указанный срок собрать?
- **Privilege:** есть ли privileged документы (адвокатская тайна, банковская тайна)?
- **ПДн:** содержат ли запрашиваемые материалы ПДн третьих лиц? (Может требовать
  санитизации)

### Шаг 4. Privilege / objections framework

#### Адвокатская тайна (ФЗ-63 ст.8)

Только для адвокатов ФПА. **Сведения, ставшие известными в связи с оказанием
юр.помощи** — не подлежат disclosure.

**Исключения** — следствие по уголовному делу с санкцией суда (ст.182 УПК).

#### Банковская тайна (395-1 ст.26)

Сведения банковских операций — disclosure только определённым органам по
определённым основаниям (suд, налоговая по конкретному запросу, следствие по
конкретным делам).

#### Налоговая тайна (НК ст.102)

Сведения от налогоплательщиков — disclosure ограничен.

#### Коммерческая тайна (98-ФЗ ст.6)

Не основание для отказа гос.органу. Но органы обязаны соблюдать confidentiality
полученных сведений.

#### ПДн третьих лиц (152-ФЗ)

Сведения о ПДн третьих лиц — должны быть санитизированы перед передачей где
возможно.

### Шаг 5. Compliance план

```yaml
- request_id: REQ-2026-001
  received_at: 2026-05-18
  source: "Арбитражный суд г. Москвы"
  basis: "АПК ст.66"
  deadline: 2026-05-28  # 10 дней
  what_requested:
    - "Копии всех договоров с ООО X за 2023-2024"
    - "Bookkeeping records операций с этим контрагентом"
    - "Email переписка по execution договоров"
  scope_analysis:
    concrete: medium  # некоторые "все документы" формулировки
    relevant: yes  # связь с предметом дела установлена
    burden: medium  # ~3 дня на full сбор
    privilege_concerns:
      - "Email с adv ФПА — адвокатская тайна — exclude"
    pdn_concerns:
      - "Bookkeeping содержит ПДн физлиц-представителей — sanitize"
  response_strategy:
    - "Provide договоры и acts (relevant + clear)"
    - "Provide bookkeeping records с санитизированными ПДн"
    - "Объявить privilege на email с адвокатом, provide общий обзор без content"
    - "Запросить продление срока на 5 дней (если burden high)"
```

### Шаг 6. Draft response

```markdown
[Адресат — суд / орган]

В ответ на [тип запроса] от [DD.MM.YYYY] № [N], направляем следующие документы и
сведения:

## Предоставляется (приложения)
1. [Документ 1] — [N листов]
2. [Документ 2] — [N листов]

## Сведения, защищённые адвокатской тайной (ФЗ-63 ст.8)
По следующим документам ссылаемся на адвокатскую тайну. Список:
- [Категория]

Просим суд исключить эти документы из истребования или рассмотреть в закрытом
режиме с участием представителей.

## ПДн третьих лиц
В предоставленных bookkeeping records ПДн физлиц-представителей контрагентов
обезличены в соответствии с 152-ФЗ. Полные данные могут быть предоставлены по
дополнительному запросу с указанием основания.

## Просьба о продлении (если применимо)
В связи с объёмом запрашиваемых материалов просим продлить срок предоставления
на [N] дней до [DD.MM.YYYY].

С уважением,
[Должность подписанта]
[ФИО]
МП
```

### Шаг 7. Calendar + tracking

- Add deadline в calendar
- Add в matter history (`/litigation:matter-update`)
- Если не успеваем — submit motion на продление **до** срока

## Specific scenarios

### Запрос следователя с обыском (УПК ст.182)

**Это серьёзная ситуация** — escalate immediately:
- CEO + GC + внешний адвокат (criminal)
- Документы изъятся следователем — нельзя предотвратить
- Что можно: ensure correct процедура (постановление о обыске должно быть
  legal), документ изъятия / inventory
- Адвокат должен быть present (право обвиняемого / подозреваемого)

### Запрос ФНС в рамках ВНП (НК ст.93)

- 10 дней default
- Можно запросить продление по объективным причинам
- Документы — first priority (без них ФНС может сделать выводы based on допущениях
  не в нашу пользу)

### Запрос ФАС (135-ФЗ)

- Возможные objections: scope too broad, нарушение порядка проверки 248-ФЗ
- Может lead к ФАС расследованию — escalate к compliance
