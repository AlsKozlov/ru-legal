# /pretensia-intake (intake досудебной претензии)

## Назначение

В РФ **обязательный претензионный порядок** по АПК ст.4 ч.5 для most commercial
disputes — без претензии иск не примут. **30 дней** на ответ (если иное не
установлено).

Этот skill — pre-drafting intake. Собирает factual + strategic context перед draft'ом
самой претензии.

## Workflow

### Шаг 1. Identification

- **Адресат претензии:**
  - Полное наименование контрагента + ИНН
  - Юридический адрес (для отправки)
  - Контактное лицо (если known)

- **Наши реквизиты:** auto-pull из company profile

- **Договор / основание:**
  - Номер договора + дата подписания (или другое основание — деликт, неосновательное
    обогащение, и т.д.)
  - Приложения / акты — relevant документы

- **Тип нарушения:**
  - `non-payment` — не оплачен товар / услуга
  - `non-performance` — не исполнено обязательство (не поставка, не работа)
  - `late-performance` — просрочка исполнения
  - `non-conformity` — товар / работа не соответствует требованиям
  - `breach-of-warranty` — нарушение гарантии
  - `restitution` — неосновательное обогащение (ст.1102 ГК)
  - `damages` — возмещение убытков (ст.15 + 393 ГК)
  - `other`

### Шаг 2. Факты

Что произошло, в хронологическом порядке. **С опорой на documents** (не "по
памяти").

Если есть chronology output от `/litigation:chronology` — use that. Иначе extract из
provided docs.

Минимум для претензии:

1. Когда заключён договор / возникло обязательство
2. Какое обязательство нарушено (с reference на пункт договора / норму ГК)
3. Когда наступил срок исполнения
4. Что фактически произошло (или не произошло) вместо исполнения
5. Какой ущерб / потери понесла наша сторона
6. Уведомления / переписка — были ли уже attempts to резolve?

### Шаг 3. Правовое обоснование (предварительное)

- Применимые статьи ГК (общая часть + специальная — для типа договора)
- Применимая судебная практика (ПП ВС РФ если есть relevant)
- Расчёт требований:
  - **Основной долг** (если non-payment) — сумма
  - **Проценты ст.395 ГК** — если денежное обязательство (расчёт по ключевой ставке ЦБ)
  - **Договорная неустойка** — по пункту договора (с учётом риска снижения по
    ст.333)
  - **Убытки** — реальный ущерб + упущенная выгода (если можем доказать)

### Шаг 4. Желаемый результат

- **Что просим в претензии:**
  - Заплатить сумму X в течение Y дней
  - Поставить товар / выполнить работу
  - Устранить недостатки
  - Возместить убытки
  - Расторгнуть договор + вернуть всё полученное

- **На что готовы settle:**
  - Compromise сумма
  - Поэтапная выплата
  - Замена обязательства (например, vendor предоставит сервис вместо денег)

### Шаг 5. Стратегический блок

Critical для не отправки "автоматической" претензии:

- **Leverage:** какие рычаги у нас?
  - Залог / обеспечение
  - Зависимость counterparty от нас (ongoing service)
  - Reputational risk для них (публичная компания, госконтракты)
  - Уголовный риск (ст.159 УК если есть признаки мошенничества)

- **BATNA** (Best Alternative To Negotiated Agreement):
  - Если не settling — что? Иск в АС? Сколько это будет cost?
  - Время — мы можем ждать или нужно сейчас?

- **Downside:**
  - Что мы теряем если претензия не сработает? (timing — 30 дней до иска)
  - Что мы теряем если иск проиграем? (госпошлина, judgement costs)
  - Counterparty счёт-фактуры могут retaliate (counter-claim)?

- **FRE 408 РФ analog (переговорный контекст):**
  - В РФ нет точного analog'а FRE 408, но переговорные коммуникации обычно не
    привлекаются в качестве доказательств (good faith negotiations)
  - Помечать тон претензии — open to negotiation vs hard line

### Шаг 6. Risk assessment

- Шансы success: high / medium / low
- Expected timeline: pretensia 30 дней + suit X месяцев
- Total cost (внутренний time + outside counsel + госпошлина)
- Expected recovery (NPV из range расчёта × шансы)

### Шаг 7. Output

```yaml
- pretensia_id: P-2026-001
  type: non-payment
  counterparty: "ООО Дебитор"
  counterparty_inn: "7707012345"
  basis:
    type: contract
    contract_number: "PD-42"
    contract_date: 2024-03-15
  facts_summary: "Не оплачена поставка на 5.25 млн руб по накладной № 100 от 01.06.2024"
  legal_basis:
    - "ГК ст.506-524 (поставка)"
    - "ГК ст.486 (оплата товара)"
    - "ГК ст.395 (проценты)"
    - "Договор п.5.2 (неустойка)"
  claim:
    principal_rub: 5250000
    interest_395_rub: 875432
    penalty_rub: 1218500  # с риском снижения ст.333
    total_rub: 7343932

  settle_floor: 5500000  # готовы на сумму без неустойки

  batna: "Иск в АС г.Москвы, ~6-8 мес, госпошлина ~50K"

  risk: high  # high chance success
  next_step: "Drафт через /litigation:pretensia-draft"
```

После intake — recommend:

> "Готов перейти к /litigation:pretensia-draft? Используем этот intake как input."

## Что НЕ делает

- Не draft'ит саму претензию (это `/pretensia-draft`)
- Не решает negotiate ли — только prep для decision
- Не оценивает шансы юридически — это lead attorney
